## Glossary

- **Calldata**: Hexadecimal-encoded function call data appended to Ethereum transactions, specifies smart contract function and parameters, critical for verifying transaction intent beyond UI display
- **Hardware wallet**: Physical device (Ledger, Trezor) storing private keys offline, requires manual transaction confirmation on device screen to prevent remote compromise
- **Raw payload**: Unformatted transaction hex data including to address, value, calldata, gas parameters, represents ground truth of transaction intent vs UI interpretation
- **Transaction simulation**: Pre-execution analysis (Tenderly, Alchemy) predicting transaction outcome including state changes, token transfers, contract interactions before actual signature
- **UI tampering**: Attack vector where malicious actors compromise wallet interface frontend (web, browser extension, hardware display) to display incorrect transaction details while embedding malicious payloads
- **Safe{Wallet}**: Web interface for Gnosis Safe multisig protocol, 80%+ market share, 5M+ ETH secured, February 2025 compromised in Bybit $1.4B hack
- **Supply chain attack**: Compromise of software development/distribution pipeline (npm packages, GitHub repos, CI/CD) to inject malicious code into wallet infrastructure
- **Alert fatigue**: Security phenomenon where excessive warnings (especially with high false positive rates) lead users to ignore alerts, undermining detection effectiveness
- **Block explorer**: Web interface (Etherscan, blockchain-specific) for inspecting on-chain transaction data, used for independent verification of raw payload contents
- **Multisig signer**: Individual authorized to approve transactions in M-of-N configuration, primary human vulnerability point for UI tampering attacks
