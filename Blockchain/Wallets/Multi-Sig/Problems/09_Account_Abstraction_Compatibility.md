## Glossary

- **Account abstraction (AA)**: Ethereum upgrade transforming accounts from private-key-controlled EOAs into smart contracts with programmable validation logic, enabling gas sponsorship, batching, recovery, custom signatures
- **EIP-4337**: Account abstraction standard avoiding consensus changes via UserOperation mempool, bundlers, paymasters, entry point contract, 40.5M accounts deployed 2024
- **EIP-7702**: Pectra upgrade allowing EOAs to temporarily execute smart contract code via delegatecall, enables gradual AA adoption without full migration
- **ERC-7579**: Modular smart account standard defining interoperable validators, executors, hooks, fallback handlers, reducing AA implementation fragmentation
- **EOA (Externally Owned Account)**: Traditional Ethereum account controlled by private key (ECDSA secp256k1), all validation via signature verification, no programmable logic
- **UserOperation**: EIP-4337 pseudo-transaction object containing AA account call data, signature, gas limits, paymaster data, processed by bundlers into actual transactions
- **Bundler**: EIP-4337 infrastructure node collecting UserOperations from mempool, validating, aggregating into single transaction submitted to entry point contract
- **Paymaster**: EIP-4337 contract sponsoring gas fees for UserOperations, enables gas-free transactions for users, requires deposit and fraud prevention mechanisms
- **Entry point**: EIP-4337 singleton contract processing bundler-submitted UserOperations, validating signatures, executing calls, managing paymasters, deployed at canonical address per chain
- **EIP-1271**: Contract signature validation standard allowing smart contracts to validate signatures, essential for AA multisig compatibility with DeFi protocols expecting EOA signatures
- **Threshold signature**: Cryptographic scheme requiring M-of-N parties to cooperate to produce valid signature, traditional multisig implementation vs AA programmable validation
- **Smart account**: Account abstraction-compliant contract wallet with programmable validation, recovery, authorization logic, vs EOA private-key-only control
- **Validator module**: ERC-7579 component defining signature validation logic for smart account, enables pluggable authentication schemes
