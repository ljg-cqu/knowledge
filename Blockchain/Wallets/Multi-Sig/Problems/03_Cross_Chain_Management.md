## Glossary

- **Cross-chain bridge**: Protocol enabling asset and message transfer between blockchains (Wormhole, LayerZero, IBC), $2B+ historical hack losses, critical for multi-chain multisig asset mobility
- **ECDSA vs EdDSA**: Elliptic Curve Digital Signature Algorithm (Ethereum, Bitcoin) vs Edwards-curve Digital Signature Algorithm (Solana, Cosmos), incompatible signature schemes requiring separate key management
- **EVM-compatible chains**: Blockchains using Ethereum Virtual Machine architecture (Polygon, Arbitrum, Optimism, Avalanche, BSC), enabling consistent multisig deployment (Gnosis Safe) vs non-EVM (Bitcoin, Solana, Cosmos)
- **Gnosis Safe**: Dominant Ethereum/EVM multisig platform deployed on 10+ chains (80%+ market share), provides consistent UX within EVM ecosystem but unsupported on non-EVM chains
- **Interoperability protocol**: Cross-chain communication infrastructure (IBC, LayerZero, Wormhole, Axelar), $8B+ TVL October 2024, often governed by multisig/threshold mechanisms
- **Key management multiplication**: Multi-chain operational burden where M-of-N multisig across C chains requires managing M×N×C key configurations, 300-600% overhead vs single-chain
- **MPC (Multi-Party Computation)**: Threshold signature technology enabling unified keys across different signature schemes (ECDSA/EdDSA), addresses cross-chain key management but different trust model from explicit multisig
- **P2SH/P2WSH**: Pay-to-Script-Hash and Pay-to-Witness-Script-Hash, Bitcoin native multisig implementations at Script level, incompatible with smart-contract-based Ethereum multisigs
- **Squads Protocol**: Solana-native multisig platform (60%+ market share on Solana), demonstrates ecosystem-specific tooling fragmentation vs EVM-centric Gnosis Safe
- **TVL (Total Value Locked)**: Metric for assets secured in protocols, $8B+ in cross-chain interoperability platforms (IBC, LayerZero, Wormhole) as of October 2024
