## Glossary

- **Seed phrase**: 12-24 word mnemonic representing complete private key, generated via BIP39 standard, enables wallet recovery, single point of compromise
- **Shamir Secret Sharing**: Cryptographic technique splitting seed phrase into N shards requiring M for reconstruction, eliminates single backup vulnerability
- **HSM (Hardware Security Module)**: Physical device providing cryptographic key generation, storage, signing in tamper-resistant hardware, enterprise custody standard
- **Social recovery**: Wallet recovery mechanism using M-of-N designated guardians (friends, family, institutions) instead of seed phrases
- **MPC (Multi-Party Computation)**: Threshold signature protocol distributing key shares never existing complete on single device, alternative to traditional seed phrase model
- **Cold storage**: Private key storage offline without internet connectivity (hardware wallets, paper wallets), maximizes security but complicates accessibility
- **Hot wallet**: Private key stored in internet-connected device (software wallet, exchange), enables convenience but increases attack surface
- **Key rotation**: Periodic replacement of private keys with new keys while maintaining same wallet address/smart contract, reduces long-term exposure
- **Estate planning**: Legal documentation specifying crypto asset location, access procedures, beneficiary designation for inheritance
- **Recovery drill**: Testing backup restoration procedures to verify functionality, identify failures before emergency need
- **Time-locked recovery**: Smart contract mechanism allowing alternative recovery method after specified inactivity period (6-12 months)
- **Backup media degradation**: Physical deterioration of storage medium (paper fading, USB corruption, hard drive failure) over time
- **M-of-N threshold**: Multisig configuration requiring M signatures from N total keys (e.g., 2-of-3, 3-of-5), M-key loss makes funds irrecoverable
