Blind signing remains prevalent for NFT minting and DeFi interactions, enabling malicious contract exploitation. With $3.1B lost in H1 2025 to crypto scams (including $91M phishing attack) and $61B NFT market growing 41.9% annually, hardware wallets need enhanced display and verification capabilities by Q3 2025.

Attack vectors: malicious NFT minting contracts, fake NFT marketplaces, contract approval phishing. ## Glossary

- **ABI (Application Binary Interface)**: Standard for encoding and decoding data when interacting with smart contracts on Ethereum and EVM chains; hardware wallets need ABI to decode raw transaction data into human-readable format.
- **Blind Signing**: Security vulnerability where users approve smart contract transactions without seeing decoded contract logic, terms, or parameters; enables malicious contracts to drain funds while displaying legitimate-looking raw data.
- **Clear Signing**: Security improvement initiative by Ledger and others to convert complex smart contract data into human-readable format on hardware wallet screens via dApp-provided plugins, eliminating blind signing risks.
- **DeFi (Decentralized Finance)**: Financial applications built on blockchain smart contracts for lending, trading, yield farming, etc.; complex interactions requiring hardware wallets to display decoded contract parameters for safe approval.
- **E Ink Display**: Electronic paper display technology using low power and providing paper-like readability; used in Ledger Stax (3.7-inch) for large-format transaction and NFT display but more expensive than LCD.
- **IPFS (InterPlanetary File System)**: Decentralized storage protocol commonly used for NFT metadata and images; hardware wallets need IPFS fetch capability or host assistance to display NFT content for verification.
- **Metadata (NFT)**: Information describing an NFT including name, description, image URL, attributes, and provenance; typically stored off-chain in IPFS or centralized servers, creating verification challenges for hardware wallets.
- **NFT (Non-Fungible Token)**: Unique blockchain token representing ownership of digital or physical assets; visual verification on hardware wallet displays important for confirming correct asset before transaction approval.
- **OpenSea, Blur, Magic Eden**: Leading NFT marketplaces where users buy, sell, and mint NFTs; hardware wallet integration requires Clear Signing support to display marketplace contract terms on-device.
- **Smart Contract**: Self-executing code on blockchain defining transaction logic; hardware wallets must decode contract parameters to show users what actions they're approving (token transfers, approvals, NFT mints, etc.).
