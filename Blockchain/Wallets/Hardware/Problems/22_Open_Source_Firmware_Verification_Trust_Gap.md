## Glossary

- **Closed-Source (Proprietary) Firmware**: Software whose source code is not publicly available for inspection, modification, or verification; users must trust the manufacturer's security claims without independent verification capability.
- **Docker**: Platform for developing, shipping, and running applications in isolated containers; commonly used in reproducible build systems to ensure consistent build environments across different machines.
- **Hash Comparison**: Cryptographic verification process where the hash (unique fingerprint) of compiled firmware is compared against the official hash to ensure binary integrity and detect tampering or build differences.
- **NDA (Non-Disclosure Agreement)**: Legal contract preventing disclosure of confidential information; secure element manufacturers require hardware wallet vendors to sign NDAs, preventing full firmware source code disclosure.
- **Nix**: Declarative package manager and build system enabling reproducible builds by precisely specifying all dependencies and build environment parameters, ensuring identical outputs from the same source code.
- **Open-Source Firmware**: Software whose source code is publicly available for inspection, modification, and verification by anyone; enables independent security audits and reduces reliance on manufacturer trust.
- **Reproducible Builds (Deterministic Builds)**: Build process that produces bit-for-bit identical binary outputs when compiled from the same source code, regardless of build machine or timing; enables users to verify that distributed binaries match published source code without backdoors.
- **RISC-V**: Open-source processor instruction set architecture enabling fully transparent hardware design; emerging as alternative to proprietary secure elements for hardware wallets, eliminating NDA constraints.
- **Secure Element (SE)**: Tamper-resistant microcontroller chip designed to securely store cryptographic keys and perform sensitive operations; typically uses proprietary firmware from manufacturers like ST Microelectronics and NXP under NDA.
- **Third-Party Security Audit**: Independent evaluation of firmware/hardware security by external experts (e.g., Trail of Bits, Kudelski Security); provides trust validation for closed-source components but requires ongoing trust in auditors.
