# Air Gapped Transaction Signing UX Complexity

**Last Updated**: 2025-11-29  
**Status**: Draft  
**Owner**: Security and Product Team

## Problem Statement

1. **[Important]** Q: Hardware wallet manufacturers face critical usability-security tradeoff with air-gapped transaction signing workflows (QR codes, microSD cards) designed to eliminate USB/Bluetooth attack surface, but introducing multi-step friction (scan transaction QR, sign on offline device, scan signature QR back to computer, broadcast) causing user errors (incomplete scans, QR tampering, workflow abandonment), limiting adoption to <5% security-conscious users vs. 95% preferring convenient USB/Bluetooth despite higher malware risk, challenging hardware wallet market differentiation in $582.98M industry. Formulate a structured problem statement using the following [Input] fields.
   
   A:
   - **Brief description of the problem to be analyzed**: 
     Air-gapped hardware wallet security vs. usability crisis: devices completely disconnected from networks (Coldcard, SeedSigner, Keystone, AirGap Vault) eliminate USB/Bluetooth attack surface [Web: Supra, 2025; Web: Blog Bitbox, 2025], but QR code transaction signing workflow requires: (1) computer generates unsigned transaction → display as QR, (2) air-gapped device scans QR via camera, (3) user reviews transaction on device screen, signs with private key, (4) device displays signed transaction as QR, (5) computer scans QR, (6) broadcast to blockchain. Workflow complexity causes user errors (QR scan failures 10-30% first-attempt rate, multi-part QR confusion, signature verification skipped), transaction abandonment (30-50% users revert to USB after first air-gapped attempt), and perceived security theater (QR tampering possible if computer compromised, though requires sophisticated malware) [Web: Blog Bitbox, 2025; Web: Keystone Blog, 2025]. Need to reduce air-gapped transaction completion time from 60-180 seconds to <30 seconds (target), increase first-time success rate from 70% to >95% (target), and expand air-gapped adoption from <5% to >25% of security-conscious hardware wallet users by Q4 2026 while maintaining USB/Bluetooth convenience for mainstream 75% preferring ease over maximum security.
   
   - **Background and current situation**: 
     Air-gapped hardware wallets: devices lacking wired/wireless connectivity (USB, Bluetooth, NFC, WiFi), transmit data via QR codes or microSD cards, preventing malware on connected computer from compromising device [Web: Supra, 2025; Web: Coinbase Learn, 2025]. Security advantage: USB connection attack surface includes malicious driver exploits, DMA (Direct Memory Access) attacks, hardware keyloggers, Bluetooth/WiFi vulnerable to man-in-the-middle, eavesdropping, remote exploits; air-gapped eliminates all digital attack vectors except QR tampering [Web: Blog Bitbox, 2025; Web: Keystone Blog, 2025]. Air-gapped devices: Coldcard ($148-$248, microSD + QR), SeedSigner (~$50 DIY, QR only), Keystone Pro ($119-$169, QR only), AirGap Vault (smartphone-based, free software + old phone, QR), Ngrave Zero ($398, QR), passport by Foundation Devices ($259, QR + microSD) [manufacturer websites]. QR code transaction flow: unsigned transaction serialized to binary, encoded to QR code (or multi-part animated QR for large transactions >1KB), displayed on computer screen, air-gapped device camera scans QR reconstructing transaction data, device parses transaction (Bitcoin PSBT - Partially Signed Bitcoin Transaction, Ethereum RLP encoding, etc.), displays human-readable details (recipient, amount, fee) for user verification, user confirms signature with PIN/password, device generates digital signature, encodes signed transaction to QR code, computer webcam scans QR, computer software decodes and broadcasts to blockchain [Web: Keystone Blog, 2025; Web: SeedSigner Guide, 2025]. Workflow challenges: (1) QR scan reliability - lighting conditions, camera quality, QR size on screen affect scan success, 10-30% first-attempt failure rate requiring repositioning device [user forums]; (2) Multi-part QR complexity - transactions >1KB require animated QR sequences (10-50 frames), user must scan all frames without interruption, confusing UX [Web: Keystone Blog, 2025]; (3) Transaction verification burden - user must manually verify recipient address and amount on small device screen (128x64 to 240x320 pixels), address truncation and tiny fonts cause missed errors; (4) Workflow time - end-to-end 60-180 seconds vs. USB 10-30 seconds for simple transaction, 3-10× slower; (5) Computer requirement - air-gapped device needs companion computer/smartphone with webcam for QR scanning, cannot operate standalone (except microSD workflow for Coldcard requiring SD card reader); (6) Perceived security theater - critics argue if computer generating QR is compromised by malware, attacker can display fake transaction QR tricking user, air-gap only protects device private key not transaction integrity [Web: Blog Bitbox, 2025; Web: YouTube, 2025]. MicroSD alternative: Coldcard supports PSBT files on microSD card (computer writes unsigned transaction to SD, insert in Coldcard, sign, reinsert in computer, broadcast), avoids QR scan issues but requires SD card reader and manual file management [Web: Coldcard Docs, 2025]. Current adoption: air-gapped wallets <5% of 10M hardware wallet market (Coldcard, SeedSigner, Keystone combined <500K devices estimated), mainstream users prefer Ledger/Trezor USB convenience despite higher malware exposure [market analysis]. Use cases favoring air-gapped: high-value holdings ($100K+), institutional custody, paranoid security enthusiasts, multisig setups, cold storage (infrequent transactions), users in adversarial environments (authoritarian regimes, high-risk professions); use cases favoring USB/Bluetooth: daily transactions, DeFi interactions, mobile usage, non-technical users, <$10K holdings where convenience-security tradeoff acceptable.

## Glossary

- **Air-Gapped**: Device completely disconnected from network (USB, Bluetooth, WiFi, NFC), transmits data via QR codes or microSD cards, eliminates digital attack surface for maximum security.
- **BBQr (Better Bitcoin QR)**: Animated QR code protocol for transactions >1KB, uses fountain codes enabling partial scan recovery, standardized format across Coldcard, Keystone, SeedSigner, Passport.
- **DMA (Direct Memory Access)**: Hardware attack enabling peripheral device to read/write system memory bypassing CPU, potential USB exploit vector for hardware wallets, mitigated by air-gap.
- **Fountain Code**: Error correction technique enabling data reconstruction from subset of transmitted packets, used in BBQr animated QR allowing partial scans to complete transaction without resending all frames.
- **Multi-Part Animated QR**: Sequence of QR codes (10-50 frames) displayed in succession to transmit large data (>1KB transactions), requires air-gapped device to scan all frames without interruption.
- **PSBT (Partially Signed Bitcoin Transaction)**: Bitcoin standard format (BIP174) for unsigned transactions enabling offline signing, used by Coldcard microSD workflow and other air-gapped wallets.
- **QR Code (Quick Response Code)**: 2D barcode encoding up to 3KB data, used by air-gapped wallets to transmit unsigned transactions (computer to device) and signed transactions (device to computer) via camera scanning.
- **Security Theater**: Measures appearing effective but providing minimal actual protection, criticism of air-gapped wallets if computer generating QR is compromised (malware displays fake transaction), though air-gap does protect device private key.
- **MicroSD Workflow**: Alternative air-gapped method using SD card (Coldcard PSBT files), computer writes unsigned transaction to SD, device signs, reinsert in computer to broadcast, avoids QR scan issues but requires card reader.
- **Truncated Address**: Displaying only first/last characters of cryptocurrency address on small screen (e.g., "1A2b...Xy9z"), user verification burden ensuring full address correctness, error-prone leading to wrong recipient.
