## Glossary

- **Seed Phrase (Recovery Phrase)**: 12-24 word mnemonic (BIP39 standard) representing private key, written on paper/metal during hardware wallet setup, enables recovery if device lost/stolen/damaged, must be kept secret and backed up securely (if lost, assets irrecoverable).
- **Emergency Asset Transfer**: Rapid procedure to move all cryptocurrency holdings from compromised wallet to new secure wallet when device lost/stolen, requires seed phrase import into temporary software wallet, transaction batching across multiple blockchains, priority fees for fast confirmation.
- **Hot Wallet**: Software-based cryptocurrency wallet (MetaMask, Trust Wallet, Exodus) storing private keys in software on computer/smartphone, used as temporary wallet during emergency recovery, ~5× higher theft risk vs. hardware wallets (vulnerable to malware, keyloggers, clipboard hijacking).
- **Seed Phrase Backup**: Physical or digital copy of 12-24 word recovery phrase stored separately from hardware wallet, recommended multi-location strategy (home safe + bank deposit box + encrypted cloud), metal plates (Cryptosteel, Billfodl) for fire/water resistance, critical for emergency recovery.
- **Crypto Wallet Insurance**: Specialized insurance policies covering cryptocurrency loss/theft from hardware wallets, premiums 1-3% annual asset value, coverage limits $500K-$5M typical, providers include Boost Insurance, Lloyd's of London cyber insurance, B2BinPay Insurance, <5% user adoption (high cost, nascent market).
- **Shamir Secret Sharing**: Cryptographic method splitting seed phrase into N shares (e.g., 5 shares), recoverable with M shares (e.g., 3-of-5), user distributes shares to multiple locations (home, bank, trusted family), if 1-2 shares lost, still recovers with remaining 3, supported by Trezor Model T/Safe 5, reduces single point of failure vs. traditional 24-word backup.
- **Multi-Location Backup**: Strategy storing seed phrase copies in multiple physical locations (home safe, bank safe deposit box, trusted family member's safe) ensuring accessibility if primary location inaccessible during emergency (travel, natural disaster, locked out of home).
- **Panic Mode UI**: User interface design optimized for high-stress emergency situations (large buttons, calming language "Your assets are safe", step-by-step guided workflow, minimal decisions required), reduces user mistakes during panic vs. standard UI assuming calm rational interaction.
- **One-Click Transfer**: Proposed emergency response feature where user pre-configures backup wallet during device setup, if device lost/stolen, single button in mobile app transfers all assets from compromised wallet to backup wallet automatically, reduces emergency response time from 15-45min to <5min.
- **Custodial Backup**: Controversial approach where seed phrase encrypted and split across custodians (Ledger Recover: Ledger, Coincover, EscrowTech), user recovers with 2-of-3 identity verification, trades self-custody principle for convenience, $9.99/mo subscription, <10% adoption due to centralization concern.
- **Blockchain Transaction Accelerator**: Service expediting cryptocurrency transaction confirmation through priority fee optimization (Bitcoin transaction accelerators, Ethereum flashbots), critical during emergency when user must confirm asset transfer before attacker (race condition), time-sensitive coordination.
