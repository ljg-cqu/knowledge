## Glossary

- **Distributed governance**: Multi-party approval system including external validators (auditors, governance councils, third-party custodians) beyond single organization's employees, achieving true decentralization of control.
- **Hardware security key**: Physical authentication device (YubiKey, Titan) providing cryptographic proof of identity, resistant to phishing and remote attacks but vulnerable to physical theft.
- **Insider threat**: Security risk from individuals within organization (employees, contractors, partners) who have authorized access but abuse privileges for malicious purposes or cause unintentional harm through errors.
- **Just-in-time (JIT) privilege elevation**: Access control model granting elevated privileges only when needed for specific tasks and limited time duration, reducing persistent privileged account attack surface.
- **Multi-party approval**: Transaction authorization requiring signatures from multiple independent parties (e.g., 3-of-5, 4-of-7), preventing single-point compromise.
- **PAM (Privileged Access Management)**: Security system controlling and monitoring privileged accounts with elevated access to critical systems, implementing just-in-time elevation and comprehensive audit logging.
- **SIEM (Security Information and Event Management)**: Centralized platform collecting, analyzing, and alerting on security events from all systems, enabling detection of unauthorized access and privilege abuse.
- **Zero-trust architecture**: Security model requiring verification for every access request regardless of source (internal or external), assuming breach and enforcing least-privilege access.
