# Biometric Authentication Privacy and Security Trade-Offs in Custody Wallets

**Status**: Draft  
**Owner**: Security Team

## Problem Statement

1. **[Important]** Q: Custody wallet providers implementing biometric authentication to improve user experience face critical privacy and security trade-offs, including vulnerability to spoofing attacks (photos, masks, 3D models), irreversible biometric data breaches with GDPR compliance challenges, potential false positives/negatives affecting access reliability, and centralized biometric template storage creating attractive attack targets, requiring careful balance between usability gains and new risk vectors. Formulate a structured problem statement using the following [Input] fields.
   
   A:
   - **Brief description of the problem to be analyzed**: 
     Biometric authentication in custody wallets creates complex security-privacy trade-offs affecting 20-30% of wallet implementations exploring biometric features, introducing spoofing vulnerabilities (photos, masks, deepfakes bypass authentication in 15-25% of tested attacks), permanent biometric data breach consequences (unlike passwords, fingerprints cannot be reset), GDPR Article 9 special category data compliance requirements, and false acceptance/rejection rates (0.1-2% FAR, 1-5% FRR) impacting legitimate user access [Web Search: Multiple Sources, 2025]. Need to develop secure biometric implementation framework achieving <0.01% spoofing success rate, zero biometric template storage on centralized servers, and <0.1% false rejection rate while meeting GDPR requirements within 12-18 months.
   
   - **Background and current situation**: 
     Biometric authentication offers improved security and user convenience over passwords, particularly with multimodal authentication combining multiple biometric factors [Web Search: OneLogin, 2025]. However, critical risks exist: (1) Spoofing vulnerabilities: Biometric systems can be bypassed using photos, masks, 3D models, with documented vulnerabilities in Windows Hello, ZKTeco terminals, and Suprema Biostar 2 systems [Web Search: Software Secured, 2025]; (2) Irreversibility: Unlike passwords, compromised biometric data (fingerprints, facial templates, iris patterns) cannot be changed or reset, creating permanent identity theft risk [Web Search: OneLogin, 2025]; (3) Privacy concerns: GDPR Article 9 classifies biometric data as "special category" requiring explicit consent, purpose limitation, and strict storage controls [Web Search: arxiv.org, 2025]; storing biometric data on blockchain creates immutability challenges conflicting with GDPR "right to erasure" [Web Search: arxiv.org, 2025]; (4) Accuracy limitations: False positive rates (FAR) of 0.1-2% mean unauthorized access risk, while false negative rates (FRR) of 1-5% mean legitimate users denied access, especially problematic for irreversible blockchain transactions [Web Search: OVIC, 2025]; (5) Centralized storage risk: Many implementations store biometric templates in centralized databases creating single attractive targets for attackers vs. decentralized custody philosophy [Web Search: Software Secured, 2025]; (6) Ethical concerns: Facial recognition systems show bias across demographic groups, raising fairness and discrimination concerns [Web Search: OneLogin, 2025]; (7) EBA Strong Customer Authentication (SCA): European Banking Authority guidance requires crypto exchanges and wallet providers to reevaluate security protocols, increasing regulatory pressure for biometric implementation while maintaining security standards [Web Search: Facephi, 2025]. Current adoption: ~20-30% of mobile wallet implementations offer biometric unlock features, primarily for convenience layer over underlying key management rather than replacing cryptographic key control [Industry Estimate: 2025].

## Glossary

- **Biometric template**: Mathematical representation of biometric data (fingerprint, facial features) used for matching; compromise risk as cannot be reset like passwords.
- **BIPA (Illinois Biometric Information Privacy Act)**: Strictest US state biometric privacy law requiring explicit consent, written policy, and limited retention for biometric data collection.
- **EBA (European Banking Authority)**: EU agency issuing Strong Customer Authentication (SCA) guidance requiring crypto services to implement robust authentication including potential biometric methods.
- **FAR (False Acceptance Rate)**: Percentage of unauthorized users incorrectly authenticated; target <0.1% for custody wallets to prevent unauthorized access to irreversible blockchain transactions.
- **FIDO2/WebAuthn**: Industry-standard protocols for passwordless authentication using public key cryptography and biometrics; enables local biometric processing without server-side template storage.
- **FRR (False Rejection Rate)**: Percentage of legitimate users incorrectly denied access; target <0.5% to prevent user frustration and access failures to digital assets.
- **GDPR Article 9**: EU regulation clause classifying biometric data as "special category" requiring explicit consent, purpose limitation, data minimization, and heightened security measures.
- **Liveness detection**: Anti-spoofing technique distinguishing live biometric presentation from photos, masks, replays; active (user action) or passive (presentation analysis) methods.
- **Multimodal biometrics**: Authentication using multiple biometric factors (e.g., facial + fingerprint + voice) to reduce spoofing risk and improve accuracy.
- **Spoofing attack**: Biometric authentication bypass using fake biometric presentations (photos, 3D masks, synthetic fingerprints, deepfake videos) to gain unauthorized access.
- **Tokenized biometrics**: Privacy-enhancing technique generating revocable tokens from biometric templates; if compromised, token can be revoked and regenerated unlike raw biometric data.
- **Zero-knowledge biometric proof**: Cryptographic method enabling biometric authentication without revealing biometric templates to verifier; strong privacy but computational overhead challenges.
