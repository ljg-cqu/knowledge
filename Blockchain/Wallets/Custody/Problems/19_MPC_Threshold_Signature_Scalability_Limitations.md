## Glossary

- **BLS12-381**: Pairing-friendly elliptic curve used for BLS signatures in Ethereum 2.0, enabling signature aggregation and threshold schemes.
- **Byzantine fault tolerance**: System's ability to function correctly even when some participants behave maliciously or unpredictably (Byzantine failures).
- **CGGMP21**: Threshold ECDSA protocol (2021) providing UC security with 3-round keygen and 1-round online signing, current institutional standard.
- **Distributed key generation (DKG)**: Cryptographic protocol where multiple parties jointly generate key shares without any party learning complete private key.
- **DKLS23**: 2-party ECDSA threshold protocol (2023) optimized for low latency, deployed in production by Coinbase for custodial wallets.
- **ECDSA (Elliptic Curve Digital Signature Algorithm)**: Signature algorithm used by Bitcoin and Ethereum based on secp256k1 elliptic curve.
- **Ed25519**: Elliptic curve signature scheme used by Solana, Cardano, and other modern blockchains for fast verification.
- **MPC (Multi-Party Computation)**: Cryptographic technique enabling multiple parties to jointly compute function over inputs while keeping inputs private.
- **Threshold signature scheme (TSS)**: Cryptographic scheme where subset (t-of-n) of parties can collectively generate valid signature without reconstructing private key.
- **UC-secure**: Universally Composable security model providing strongest security guarantees for cryptographic protocols under composition.
