## Glossary

- **CDN (Content Delivery Network)**: Distributed network of servers delivering web content with low latency and DDoS protection, single point of failure when platforms depend on one provider (Cloudflare, Akamai, Fastly).
- **Chaos engineering**: Discipline of experimenting on distributed systems to build confidence in resilience, using tools like Gremlin and Chaos Monkey to simulate failures proactively.
- **DePIN (Decentralized Physical Infrastructure Networks)**: Blockchain-based networks incentivizing individuals to provide infrastructure (storage, compute, bandwidth) as alternative to centralized clouds.
- **DNS failover**: Technique redirecting traffic to backup infrastructure when primary provider fails, using health checks and automatic DNS record updates.
- **IPFS (InterPlanetary File System)**: Decentralized, peer-to-peer file storage and content delivery protocol, alternative to centralized CDNs with 2-5x latency penalty.
- **Mean Time To Recover (MTTR)**: Average time to restore service after outage, currently 2-8h for infrastructure provider failures without automated failover.
- **Multi-CDN strategy**: Architecture using multiple CDN providers with automatic failover, reducing single-point-of-failure risk but adding 30-50% cost and operational complexity.
- **Single point of failure**: System component whose failure causes entire service outage, exemplified by November 2025 Cloudflare incident disrupting 20% of internet including major crypto platforms.
