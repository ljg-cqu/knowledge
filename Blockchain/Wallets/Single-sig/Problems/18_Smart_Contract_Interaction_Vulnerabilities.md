## Glossary

- **Access control vulnerability**: Smart contract flaw where ownership checks, role management, or permission enforcement missing/incorrect, enabling unauthorized functions; caused $1.6B losses H1 2025 (70% of $2.3B total exploits).
- **Approval revocation**: Process of revoking previously-granted token spending permissions by calling `approve(spender, 0)` or `setApprovalForAll(operator, false)`; requires manual action via Revoke.cash or block explorer, <5% user adoption.
- **Delegatecall vulnerability**: Critical smart contract flaw where `delegatecall` executes external code in calling contract's storage context, enabling attackers to manipulate state variables and steal funds; OWASP Smart Contract Top 10 vulnerability.
- **EIP-712**: Ethereum standard for structured message signing enabling human-readable signatures for dApp interactions; improves over raw signatures but exploited for phishing when users don't verify structured data content.
- **Fire.com / Blowfish**: Real-time transaction security scanning services analyzing smart contract calls before execution to detect malicious behavior; <10% wallet integration limiting effectiveness.
- **Inferno Drainer**: Scam-as-a-Service (ScaaS) platform enabling non-technical attackers to deploy wallet drainer campaigns; stole $5.9M+ across multiple operations; exemplifies democratization of smart contract attacks lowering technical barriers.
- **Permit signature**: Gasless token approval using EIP-2612 enabling off-chain signatures instead of on-chain `approve()` transactions; exploited by drainers to bypass standard transaction warnings and grant unlimited spending rights.
- **Phishing signature attack**: Social engineering tricking users into signing malicious transactions through fake dApp interfaces (counterfeit NFT mints, phishing DEX sites); reached $790M cumulative losses, 31% of all attacks 2024.
- **Reentrancy attack**: Smart contract vulnerability enabling attacker to recursively call victim contract before state updates complete, draining funds; famous example: TheDAO hack $50M (2016); OWASP Smart Contract Top 10 vulnerability.
- **Revoke.cash**: Third-party service enabling one-click token approval revocation through web interface; reduces wallet drainer risk but requires user awareness and manual action, <5% adoption.
- **ScaaS (Scam-as-a-Service)**: Platforms (Inferno Drainer, Pink Drainer, Monkey Drainer) enabling non-technical attackers to deploy phishing/drainer campaigns through ready-made infrastructure; democratized attacks causing $494M losses 2024.
- **Smart contract audit**: Security review by specialized firms (CertiK, Quantstamp, Trail of Bits, OpenZeppelin) analyzing contract code for vulnerabilities; costs $20K-$200K per contract; only 15-25% of DeFi protocols audited; doesn't guarantee security (audited contracts still exploited).
- **Transaction simulation**: Wallet feature executing transaction in sandboxed environment before actual submission to preview outcomes (token transfers, approval grants, state changes); adoption <10% of wallets; sophisticated attacks evade through simulation spoofing.
- **Transaction simulation spoofing**: Emerging attack technique displaying false transaction preview to users by manipulating simulation response, bypassing wallet security warnings; identified by ScamSniffer 2024 as evolving threat.
- **Unlimited token approval**: Standard DeFi UX pattern granting smart contracts permission to spend infinite tokens from user's wallet using `approve(spender, type(uint256).max)` for convenience; catastrophic if contract malicious; 85% of DeFi interactions use unlimited approvals.
- **Wallet drainer**: Malicious smart contract or dApp frontend tricking users into signing transactions granting token approvals or direct transfers, immediately draining assets; caused $494M losses across 332,000 addresses in 2024 (+67% YoY), $800M+ cumulative since 2023.
