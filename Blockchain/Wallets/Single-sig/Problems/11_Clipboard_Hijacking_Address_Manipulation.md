## Glossary

- **Address poisoning**: Attack where malicious actors send zero-value transactions from lookalike addresses (matching first 6 and last 4 characters of legitimate address) to pollute victim's transaction history, tricking them into copying the malicious address later.
- **Checksum**: Cryptographic hash embedded in cryptocurrency address format that allows detection of typos or manipulations; address validation algorithms verify checksum integrity before transaction submission.
- **Clipper malware (clipboard hijacker)**: Malware that monitors operating system clipboard for cryptocurrency addresses and replaces them with attacker-controlled addresses, redirecting funds to attackers when victims paste addresses.
- **Identicon**: Visual representation of cryptocurrency address as unique geometric pattern or avatar, enabling quick visual verification that address hasn't changed between copy and paste operations.
- **Lookalike address**: Cryptocurrency address generated to resemble legitimate address, typically matching first 6 and last 4 characters (vanity address generation), used in address poisoning attacks.
- **QR code**: Two-dimensional barcode encoding cryptocurrency address, eliminating clipboard attack vector by encoding address directly from sender's screen to recipient's camera without clipboard interaction.
- **Single-signature wallet (single-sig)**: Cryptocurrency wallet requiring only one private key to authorize transactions, lacking multi-party verification that could detect address manipulation.
- **Vanity address**: Cryptocurrency address with custom prefix or suffix matching specific pattern, generated through brute-force mining; legitimate use for branding, malicious use for lookalike addresses.
- **Warning fatigue**: Psychological phenomenon where users habitually dismiss security warnings without reading them, reducing effectiveness of alert-based security controls.
- **Zero-value transaction**: Blockchain transaction transferring 0 cryptocurrency (only paying gas fees), used in address poisoning to create transaction history entries without actual value transfer.
