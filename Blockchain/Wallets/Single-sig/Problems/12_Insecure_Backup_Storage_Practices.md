## Glossary

- **BIP-39 (Bitcoin Improvement Proposal 39)**: Standard defining how seed phrases (mnemonic words) are generated and used to derive cryptocurrency private keys.
- **Client-side encryption**: Encryption performed on user's device before data uploaded to cloud, preventing cloud provider from accessing plaintext; requires user-controlled encryption keys.
- **Cryptosteel/Billfodl**: Commercial metal backup devices allowing users to physically stamp seed phrase words into stainless steel, resistant to fire (up to 1500°C), water, and corrosion.
- **EXIF metadata (Exchangeable Image File Format)**: Data embedded in photo files including timestamp, GPS coordinates, device model, camera settings; security risk when seed phrase photos contain location data.
- **OCR (Optical Character Recognition)**: Technology converting images of text into machine-readable text; used by malware to extract seed phrases from screenshots.
- **Password manager**: Application storing passwords and sensitive data in encrypted vault, synchronized across devices; examples include LastPass, 1Password, Bitwarden.
- **Screenshot**: Digital image capture of device screen; common insecure backup method for seed phrases, vulnerable to cloud sync, malware theft, and metadata exposure.
- **Seed phrase (mnemonic phrase)**: Sequence of 12-24 words used to generate and recover cryptocurrency wallet private keys; losing or exposing seed phrase results in permanent loss or theft of funds.
- **Shamir's Secret Sharing**: Cryptographic algorithm dividing seed phrase into multiple shares, where threshold subset (e.g., 3 of 5) can reconstruct original; increases security and redundancy but adds complexity.
- **SpyAgent malware**: Android malware targeting cryptocurrency wallets by stealing screenshots from device gallery, using OCR to extract recovery phrases, and uploading to attacker servers.
- **Zero-knowledge architecture**: System design where service provider cannot access user data even if servers compromised; encryption keys held exclusively by user.
