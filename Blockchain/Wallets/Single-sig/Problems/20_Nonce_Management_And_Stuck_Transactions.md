## Glossary

- **Account abstraction (ERC-4337)**: Ethereum standard (March 2023) enabling programmable smart contract wallets with features like gas sponsorship, automated transaction retries, batching, and custom validation logic; <2% wallet adoption as of 2025 but growing; streamlines UX reducing stuck transactions.
- **Base fee**: EIP-1559 component of Ethereum gas pricing; algorithmically-determined minimum fee per gas unit burned (not paid to validators); adjusts based on network congestion; reduces fee volatility but doesn't eliminate priority fee competition.
- **CPFP (Child-Pays-for-Parent)**: Bitcoin transaction acceleration method where recipient creates high-fee "child" transaction spending stuck "parent" transaction's output, incentivizing miners to include both; alternative to RBF.
- **Dencun upgrade**: Ethereum protocol upgrade (March 2024) introducing proto-danksharding (EIP-4844) significantly reducing average gas fees and L2 costs; gas volatility persists despite lower baseline.
- **EIP-1559**: Ethereum Improvement Proposal (August 2021) reforming gas market with base fee (burned) + priority fee (to validators); reduces but doesn't eliminate fee volatility; enables better fee estimation.
- **Gas estimation**: Process of predicting computational cost (gas units) required for transaction execution; wallets use `eth_estimateGas` RPC call; accuracy 70-85% due to future network state unpredictability; underestimation causes transaction failures.
- **Mempool**: Memory pool of pending blockchain transactions awaiting validator/miner confirmation; Bitcoin transactions auto-expire after 14 days default; Ethereum transactions persist indefinitely unless manually replaced; visible to public enabling MEV attacks.
- **Nonce (Number Used Once)**: Transaction ordering counter ensuring sequential execution and replay protection; Ethereum account nonce increments with each transaction (must execute in order: nonce 5 before nonce 6); incorrect nonce handling causes stuck transactions and cascade failures.
- **Priority fee**: EIP-1559 component paid to validators for transaction prioritization; users bid priority fee competing for block space; insufficient priority fee causes stuck transactions during congestion; replaces legacy "gas price" concept.
- **RBF (Replace-by-Fee)**: Bitcoin protocol feature enabling stuck transaction replacement by broadcasting new transaction with same inputs, higher fee, and different nonce; requires wallet support (<50% Bitcoin wallets support as of 2025).
- **Smart Transactions**: MetaMask feature (May 2024) aiming to reduce transaction failure rates and protect from MEV through private relay and automated retries; not default-enabled; limited to MetaMask users (20-30% wallet market).
- **Stuck transaction**: Transaction remaining unconfirmed in mempool for extended period (hours to days) due to insufficient gas fee bid relative to network congestion; blocks subsequent transactions from same address due to nonce sequencing; 30-50% users affected during congestion.
- **Transaction replacement**: Process of canceling or accelerating stuck transaction by broadcasting new transaction with same nonce and higher gas fee; MetaMask "Speed Up" (increases fee 10-20%) and "Cancel" (0-value to self) buttons implement replacement; success not guaranteed if original confirms first.
