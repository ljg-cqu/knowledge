## Glossary

- **Address Book**: Wallet feature allowing users to save and label frequently used blockchain addresses, reducing manual entry errors and poisoning risks through pre-verified recipients.
- **Address Poisoning**: Blockchain scam where attackers generate lookalike addresses mimicking legitimate ones, then flood victim's transaction history to trick them into sending funds to attacker's address.
- **Checksum Address**: EIP-55 format using mixed case (upper/lower) in Ethereum addresses to detect typos; checksum validation catches ~99.986% of transcription errors but not lookalike attacks.
- **Counterfeit Token Transfer**: Address poisoning variant where attacker creates fake ERC20 token with identical symbol to legitimate token, transferring exact amounts to mimic victim's transaction history.
- **ENS (Ethereum Name Service)**: Decentralized domain name system for Ethereum, mapping human-readable names (example.eth) to 40-character addresses; ~2M registered domains as of 2024.
- **ERC20 transferFrom()**: Token standard function allowing approved spenders to transfer tokens; exploited in poisoning attacks via zero-value transfers with arbitrary sender/receiver.
- **Hexadecimal Address**: 40-character (160-bit) blockchain address format using 0-9, a-f characters; derived from public key via Keccak-256 hash, designed for cryptographic security not human usability.
- **Irreversible Transaction**: Blockchain transaction property where once confirmed, funds cannot be recalled or reversed even if sent to wrong address, unlike traditional banking chargebacks.
- **Lookalike Address**: Maliciously generated address matching first N and last M characters of target address, exploiting user tendency to verify only prefix/suffix in wallet UIs.
- **Tiny Transfer**: Address poisoning technique sending small amounts of legitimate tokens from lookalike address to victim, inserting attacker's address into victim's transaction history.
- **Zero-Value Transfer**: Poisoning attack exploiting ERC20 tokens allowing `transferFrom(victim, lookalike, 0)` calls, creating fake transaction history showing victim "sent" to attacker's address.
