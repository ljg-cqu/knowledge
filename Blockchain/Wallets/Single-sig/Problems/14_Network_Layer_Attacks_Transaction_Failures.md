## Glossary

- **Arbitrage**: Trading strategy exploiting price differences across markets; in crypto, MEV bots execute arbitrage using user transactions as price discovery, capturing profit.
- **AWS (Amazon Web Services)**: Cloud computing platform hosting major RPC providers (Infura, Alchemy), creating single point of failure for blockchain infrastructure.
- **DEX (Decentralized Exchange)**: Blockchain-based exchange using smart contracts for peer-to-peer trading (Uniswap, Curve); vulnerable to front-running and sandwich attacks.
- **Front-running**: MEV attack where bot monitors mempool for profitable transaction, submits identical transaction with higher gas to execute first, capturing user's expected gain.
- **Infura**: ConsenSys-owned RPC provider with est. 60-70% market share of Ethereum dApp traffic; default provider for MetaMask; experienced outages and implemented Tornado Cash censorship.
- **Mempool**: Memory pool storing unconfirmed blockchain transactions awaiting inclusion in block; public visibility enables MEV extraction, vulnerable to DDoS flooding.
- **MEV (Maximal Extractable Value)**: Value extracted from users through transaction reordering, front-running, sandwich attacks, and arbitrage; averaged $300K daily on Ethereum in 2024.
- **PBS (Proposer-Builder Separation)**: Ethereum protocol change separating block construction (builders) from validation (proposers), reducing validator-MEV collusion, implemented with 2022 Merge.
- **Private mempool**: Transaction submission bypassing public mempool (Flashbots Protect, Eden Network), negotiating directly with block builders to prevent front-running.
- **RPC (Remote Procedure Call)**: Protocol allowing wallets and dApps to communicate with blockchain nodes; centralized providers (Infura, Alchemy) create dependency and censorship risks.
- **Sandwich attack**: MEV attack placing buy order before victim's trade (raising price) and sell order after (lowering price), extracting profit from victim's slippage; $215K stolen from Uniswap trader in 2025.
- **Slippage**: Price difference between expected and executed trade; users set "slippage tolerance" (e.g., 2%) allowing MEV bots to extract value within tolerance.
