## Glossary

- **ABI (Application Binary Interface)**: Interface defining how to interact with Ethereum smart contracts, specifying function signatures and parameter types; complexity prevents human readability of raw calldata.
- **Blind signing**: Approving cryptocurrency transaction without understanding full contents, typically required for complex smart contract interactions exceeding hardware wallet display capacity.
- **Calldata**: Data field in Ethereum transaction containing function call parameters sent to smart contract; often 1KB-10KB of hex-encoded data unintelligible to humans.
- **Clear signing**: Hardware wallet capability to display human-readable transaction details (e.g., "Swap 1 ETH for 2000 USDC") instead of raw hex data; limited to known contract ABIs.
- **DeFi (Decentralized Finance)**: Blockchain-based financial applications (lending, trading, derivatives) using smart contracts instead of intermediaries; complexity drives blind signing necessity.
- **Hardware wallet**: Physical device storing cryptocurrency private keys offline, requiring button press to sign transactions; provides security against software malware but vulnerable to UI manipulation.
- **Lazarus Group**: North Korean state-sponsored hacking group responsible for multiple major cryptocurrency thefts including 2025 Bybit $1.5B heist.
- **Multisig (multi-signature)**: Wallet requiring multiple private keys to authorize transactions (e.g., 3 of 5 signers); Bybit used multisig but all signers deceived by compromised UI.
- **TEE (Trusted Execution Environment)**: Secure area of processor isolating code and data from main OS; used for cryptographic operations and verified display paths (ARM TrustZone, Intel SGX).
- **Transaction simulation**: Pre-execution analysis of blockchain transaction to predict outcome (balance changes, approvals, risks) before user signs; reduces blind signing risk.
- **UI manipulation attack**: Compromise of wallet user interface to display legitimate transaction while actual signed data is malicious; exploits separation between display device and signing device.
- **WYSIWYS (What You See Is What You Sign)**: Security principle that displayed transaction details cryptographically match signed data; failure enabled Bybit attack.
