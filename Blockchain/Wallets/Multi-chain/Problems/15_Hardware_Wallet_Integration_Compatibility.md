## Glossary

- **Hardware Wallet**: Physical device storing cryptocurrency private keys offline (cold storage), requiring physical confirmation for transaction signing to prevent remote hacking.
- **Cold Storage**: Cryptocurrency storage method keeping private keys completely offline, isolated from internet-connected devices.
- **Firmware**: Low-level software embedded in hardware wallet controlling device operations, transaction signing, and blockchain protocol support.
- **WalletConnect**: Open-source protocol enabling secure connections between crypto wallets and dApps using QR code or deep link pairing, supporting 70K+ applications.
- **WebUSB/WebHID**: Browser APIs enabling web applications to communicate with USB devices (WebUSB) and human interface devices (WebHID) for hardware wallet integration without installing drivers.
- **Secure Element**: Tamper-resistant chip in hardware wallets (e.g., Ledger's CC EAL5+ certified chip) providing cryptographic operations and key storage with hardware-level security.
- **Derivation Path**: Hierarchical structure (BIP32/BIP44) defining how cryptocurrency addresses are generated from single seed phrase, varying by blockchain (e.g., m/44'/60'/0'/0 for Ethereum).
- **Multi-chain Wallet**: Software or hardware wallet supporting multiple blockchain networks and cryptocurrencies through single interface, enabling unified asset management.
- **dApp (Decentralized Application)**: Application running on blockchain networks (smart contracts) rather than centralized servers, requiring wallet connections for user interactions.
