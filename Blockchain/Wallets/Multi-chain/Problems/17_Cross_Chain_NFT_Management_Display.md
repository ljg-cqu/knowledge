## Glossary

- **NFT (Non-Fungible Token)**: Blockchain token representing unique digital or physical assets with distinct metadata, utilizing standards like ERC-721 (Ethereum unique tokens) or ERC-1155 (multi-token standard).
- **IPFS (InterPlanetary File System)**: Decentralized peer-to-peer protocol for storing and sharing files using content addressing (CIDs), commonly used for NFT metadata and media storage off-chain.
- **ERC-721**: Ethereum token standard for non-fungible tokens where each token is unique with distinct tokenId and metadata, commonly used for collectibles, art, and unique digital assets.
- **ERC-1155**: Ethereum multi-token standard supporting both fungible and non-fungible tokens within single smart contract, enabling batch transfers and reducing gas costs.
- **NFT Metadata**: JSON data describing NFT attributes (name, description, image URL, properties), typically stored on IPFS and referenced by on-chain smart contract via tokenURI.
- **IPFS Gateway**: HTTP interface enabling browsers and applications to access IPFS content without running full IPFS node, converting IPFS CIDs to HTTP URLs.
- **NFT Indexer**: Service continuously scanning blockchains to aggregate NFT ownership, transfers, and metadata into queryable database for fast wallet and marketplace displays.
- **Content Addressing**: IPFS method of referencing files by cryptographic hash (CID) of content rather than location, ensuring immutability and verifiability.
- **CDN (Content Delivery Network)**: Distributed server network caching content at edge locations for faster delivery, critical for NFT media loading performance.
