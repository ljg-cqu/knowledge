## Glossary

- **EOA (Externally Owned Account)**: Traditional blockchain account controlled by a single private key, as opposed to smart contract account with programmable logic. Most common wallet type but creates single point of failure.
- **HSM (Hardware Security Module)**: Specialized physical computing device for safeguarding and managing cryptographic keys, providing tamper-resistant storage and secure key operations.
- **MPC (Multi-Party Computation)**: Cryptographic technique enabling multiple parties to jointly compute a function over their inputs while keeping inputs private. In wallet context, enables distributed key generation and transaction signing without reconstructing full private key.
- **Multi-signature (multi-sig)**: Security mechanism requiring M-of-N private keys to authorize transactions (e.g., 2-of-3 or 3-of-5), eliminating single point of failure but adding complexity and cost.
- **Private key**: Secret cryptographic key that controls a blockchain wallet, enabling transaction signing and asset access. Compromise of private key grants full control to attacker.
- **Seed phrase (Recovery phrase)**: Typically 12 or 24 words representing a wallet's master private key in human-readable form, used for wallet backup and recovery.
- **TEE (Trusted Execution Environment)**: Secure area of processor ensuring code and data are protected with respect to confidentiality and integrity, commonly used for secure key storage on mobile devices.
- **Threshold signature (TSS)**: Cryptographic signature scheme where a threshold number of parties (M-of-N) must cooperate to create a valid signature, core technology enabling MPC wallets.
