## Glossary

- **CAIP-10**: Chain Agnostic Improvement Proposal for account addressing standard that works across different blockchain ecosystems.
- **Chain abstraction**: Technology layer that unifies multi-chain interactions, allowing users to access multiple blockchains through single interface without managing separate accounts, balances, or network switches.
- **Chain Signatures**: NEAR protocol feature (launched Aug 2024) enabling NEAR accounts to sign and execute transactions across multiple blockchains using Multi-Party Computation (MPC).
- **Derivation path**: Cryptographic method for generating multiple addresses from single seed, used in hierarchical deterministic (HD) wallets and cross-chain account systems.
- **ERC-4337**: Ethereum account abstraction standard enabling smart contract wallets with programmable features; limited to EVM-compatible chains.
- **Gas abstraction**: Technology allowing users to pay transaction fees in any token or have fees sponsored by third parties, eliminating need to hold native blockchain tokens for gas.
- **Intents**: Declarative transaction framework where users specify desired outcomes (e.g., "swap 100 USDC for ETH at best price") rather than execution steps, with system determining optimal execution path.
- **MPC (Multi-Party Computation)**: Cryptographic protocol allowing multiple parties to jointly compute function over inputs while keeping inputs private, used for distributed key management and cross-chain signing.
- **Universal Account**: Single account identity that works across multiple blockchains, providing unified balance view and transaction capabilities without managing separate chain-specific accounts.
