## Glossary

- **Alt-L1 (Alternative Layer 1)**: Non-Ethereum blockchain platforms with independent consensus mechanisms (e.g., Solana, Avalanche, BSC, Cosmos, Polkadot). Requires separate bridge infrastructure for interoperability with Ethereum ecosystem.
- **Bridge Protocol**: Infrastructure enabling asset and data transfer between different blockchains. Common types: lock-and-mint (lock assets on source chain, mint wrapped on destination), liquidity pools (swap assets across chains), light-client verification (cryptographically verify cross-chain state).
- **Chain Abstraction**: Emerging design pattern aiming to hide cross-chain complexity from users, enabling single-interface interaction with multi-chain applications without explicit bridge transactions or chain-switching. Still early-stage with competing proposals.
- **Chainlink CCIP (Cross-Chain Interoperability Protocol)**: Decentralized messaging protocol enabling secure cross-chain communication using Chainlink's oracle network. Provides risk management framework and programmable token transfers across chains.
- **Cross-Chain Messaging**: Protocols enabling smart contracts on one blockchain to trigger actions on another chain. Examples: LayerZero (ultra-light nodes), Axelar (Cosmos-based), Wormhole (guardian network), Chainlink CCIP (oracle-based verification).
- **LayerZero**: Cross-chain messaging protocol using ultra-light nodes and relayers to enable omnichain applications. Claimed security model: independent verification by oracles and relayers, with user-configurable trust assumptions.
- **Light-Client Bridge**: Cross-chain bridge using cryptographic proofs (block headers, Merkle proofs) to verify source chain state on destination chain without trusted intermediaries. Higher security but increased gas costs (300K-1M gas per message) and implementation complexity.
- **Lock-and-Mint Bridge**: Cross-chain bridge architecture locking assets on source chain via smart contract or custodian, then minting equivalent wrapped assets on destination chain. Security depends on lock mechanism (multi-sig, threshold signatures, smart contract validation).
- **Liquidity Pool Bridge**: Cross-chain bridge using pre-funded liquidity pools on multiple chains to enable instant asset swaps without minting wrapped tokens. Examples: Connext, Hop Protocol, Across. Limited by pool liquidity depth (typically top 10 tokens only).
- **Multi-Sig (Multi-Signature)**: Security mechanism requiring M-of-N signers to authorize transactions. Used by bridges for validating cross-chain messages, but vulnerable to key compromise, collusion, or insufficient geographic/organizational decentralization.
- **TVL (Total Value Locked)**: Total dollar value of crypto assets locked in DeFi protocol or bridge. Key metric for bridge security risk (higher TVL = larger attack incentive) and ecosystem health (TVL >$500M considered competitive threshold).
- **Wrapped Token**: Tokenized representation of asset from one blockchain on another chain (e.g., WBTC = Bitcoin on Ethereum, wETH = ETH on Polygon). Backed 1:1 by locked assets, but introduces custodial risk and liquidity fragmentation across multiple wrapped versions.
