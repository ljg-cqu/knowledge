## Glossary

- **90-day Wallet Retention**: Percentage of wallet users who remain active (≥1 transaction) 90 days after initial wallet creation. Current baseline 11% (89% dormancy). Critical metric for onboarding effectiveness and long-term adoption.
- **Account Abstraction (ERC-4337)**: Ethereum standard enabling smart contract wallets with programmable features (gas sponsorship, transaction batching, social recovery) improving UX. Enables gasless onboarding via paymasters. Adds conceptual complexity for developers.
- **Cognitive Overload**: Psychological state where information exceeds working memory capacity, causing confusion and abandonment. Common in Web3 onboarding due to unfamiliar concepts (gas, networks, keys, transactions) presented simultaneously. Mitigated through progressive disclosure.
- **Embedded Wallet**: Wallet integrated directly into DApp rather than separate application (MetaMask, Coinbase Wallet). Reduces onboarding friction by eliminating app switching. Examples: Privy, Dynamic, Thirdweb. Trade-off: portability and user concerns about lock-in.
- **Fiat On-ramp**: Service converting traditional currency (USD, EUR) to cryptocurrency, enabling wallet funding without existing crypto holdings. Requires KYC compliance. Friction point in onboarding (verification time, fees). Alternatives: crypto transfers, faucets, gasless transactions.
- **Gasless Transaction**: Transaction where gas fees paid by third party (paymaster) rather than user. Enabled by account abstraction (ERC-4337). Critical for onboarding (removes funding friction). Sustainability concern: who sponsors long-term?
- **Onboarding Completion Rate**: Percentage of users who start wallet creation and successfully complete all required steps (account creation, backup, funding, first transaction). Current baseline 50-70%. Target >90%.
- **Progressive Disclosure**: UX pattern revealing information gradually based on user needs rather than upfront. Shows basic features first, advanced options later. Reduces cognitive overload. Measured 15-30% improvement in onboarding completion vs. full-featured initial screens.
- **Seed Phrase**: 12-24 word mnemonic backup enabling wallet recovery. Major onboarding friction point with 30-50% abandonment during backup step. Security responsibility creates user anxiety. Alternative: social recovery, custodial models, passkeys.
- **Social Login**: Authentication using existing accounts (Google, Apple, Twitter) via OAuth. Reduces onboarding time from 15-45 minutes to 2-5 minutes. Implementations: web3auth, Magic, Privy. User confusion about custody model ("where are my keys?") and recovery dependencies on OAuth providers.
