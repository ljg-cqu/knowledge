## Glossary

- **Account Abstraction (ERC-4337)**: Ethereum standard enabling smart contract wallets with programmable transaction validation, gas sponsorship (paymasters), and custom logic without protocol changes. Separates transaction signing from execution.
- **Access Control**: Security mechanisms restricting who can execute specific smart contract functions, typically using role-based permissions (owner, admin, user roles) to prevent unauthorized operations.
- **Audit Coverage**: Percentage of deployed smart contracts that have undergone professional security audits by qualified third-party firms, measured as (audited contracts / total high-value contracts) × 100%.
- **Flash Loan Attack**: Exploit using uncollateralized loans executed and repaid within a single transaction to manipulate market prices, exploit protocol logic, or drain funds through reentrancy or price oracle manipulation.
- **Formal Verification**: Mathematical proof techniques verifying smart contract code correctness against formal specifications, providing strongest security guarantees but requiring specialized expertise and significant investment ($100K-$500K per contract).
- **Private Key**: Cryptographic secret (256-bit number) controlling wallet access; compromise enables unauthorized fund transfers. In smart contract wallets, may be used for initial authentication before on-chain logic execution.
- **Proxy Pattern**: Smart contract design enabling upgradeability by separating storage (proxy) from logic (implementation), includes UUPS (Universal Upgradeable Proxy Standard), transparent proxy, and beacon proxy variants.
- **Reentrancy Attack**: Vulnerability where external contract calls allow attacker to recursively re-enter function before state updates complete, potentially draining funds (e.g., The DAO hack - $60M loss in 2016).
- **Social Recovery**: Wallet recovery mechanism using trusted guardians (family, friends, institutions) who collectively authorize new signing key creation if primary key lost, eliminating single seed phrase dependency.
- **Unchecked External Call**: Vulnerability when contract fails to validate return values from external calls, potentially allowing silent failures or unexpected state changes that attackers exploit.
