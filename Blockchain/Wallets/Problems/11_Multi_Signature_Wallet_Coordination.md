## Glossary

- **M-of-N threshold**: Multi-signature configuration requiring M signatures from N total key holders to authorize transaction (e.g., 2-of-3, 3-of-5)
- **DAO (Decentralized Autonomous Organization)**: Blockchain-based organization with token holder governance, typically using multisig for treasury management
- **Gnosis Safe**: Leading multisig wallet platform (80%+ market share), 5M+ ETH secured, smart contract-based multi-signature implementation
- **Key holder (Signer)**: Individual or entity controlling one of N keys in multisig configuration, responsible for transaction approval
- **Key loss incident**: Loss of ≥1 keys in multisig wallet, may require wallet migration if threshold cannot be met
- **MPC (Multi-Party Computation)**: Alternative to traditional multisig using distributed key generation, eliminates on-chain coordination
- **Nonce conflict**: Transaction ordering issue when multiple multisig proposals submitted concurrently
- **Time-locked recovery**: Mechanism allowing wallet access after delay period if original key holders unavailable, security-convenience trade-off
- **Transaction simulation**: Pre-execution preview of multisig transaction outcome, reduces approval errors 40-50%
