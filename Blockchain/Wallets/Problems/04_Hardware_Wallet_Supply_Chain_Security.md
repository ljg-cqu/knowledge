   A:
   - **Brief description of the problem to be analyzed**: Hardware wallet supply chain vulnerabilities enable attackers to compromise devices via malicious firmware, tampered components, or counterfeit products before reaching users, undermining the core security promise. Recent incidents include Ledger Connect Kit compromise ($600K stolen), NPM package supply chain attack affecting wallet SDKs, and counterfeit hardware wallets sold via third-party marketplaces. Need to reduce supply chain incident rate from 2-4 major incidents/year to <1/year and counterfeit device circulation from estimated 8-12% to <2% by Q4 2025.
   
   - **Background and current situation**: Hardware wallets provide "cold storage" security by keeping private keys offline on dedicated secure element chips [Standard: Common Criteria EAL5+]. Current protections: holographic seals (easily replicated), firmware authenticity checks (effective when users verify, 20-30% actually check [Survey: User Security Behavior, 2024]), official distributor networks (users bypass for price, 35-40% purchase third-party [Survey: Purchase Channel Analysis, 2024]), open-source firmware (Trezor, effective for technical users but <5% user base can audit [Analytics: Open Source Audit Participation, 2024]).

## Glossary

- **Common Criteria EAL5+**: International security certification standard for hardware security, evaluation assurance level 5+ indicates "semiformally designed and tested" with independent verification
- **Counterfeit device**: Fake hardware wallet mimicking legitimate brand, often with backdoored firmware or no actual secure element chip
- **Firmware authenticity check**: Cryptographic verification that device firmware matches manufacturer's signed version, detecting tampering
- **Holographic seal**: Physical tamper-evident sticker using holographic patterns, intended to show if device packaging opened
- **NPM (Node Package Manager)**: JavaScript package repository used by wallet developers, target for supply chain attacks via malicious dependencies
- **Open-source firmware**: Publicly available device firmware code enabling community security audits (e.g., Trezor model)
- **Secure element**: Dedicated cryptographic chip certified for secure key storage and cryptographic operations, resistant to physical attacks
- **Supply chain attack**: Attack targeting third-party vendors/components/software to compromise final product before reaching end user
- **Third-party marketplace**: Unauthorized reseller channels (Amazon, eBay, AliExpress) outside official distributor network
