## Glossary

- **Phishing**: Fraudulent attempts to obtain sensitive information (private keys, seed phrases, passwords) by impersonating legitimate entities via fake websites, emails, or messages. Causes $880M annual crypto losses (40.8% of incidents). Detection: verify URLs, check for HTTPS, scrutinize sender addresses, never share seed phrases.
- **Social Engineering**: Psychological manipulation techniques to trick users into divulging confidential information or performing actions that compromise security. Examples: impersonating support teams, creating urgency ("your account will be locked"), offering fake rewards. Defense: verify through official channels, ignore unsolicited contact.
- **Seed Phrase (Recovery Phrase)**: 12-24 word mnemonic phrase that deterministically generates private keys for cryptocurrency wallets. Sole backup mechanism for self-custody; if lost, funds permanently inaccessible (20% of Bitcoin lost). Must be stored offline, never digitally (no screenshots, cloud storage, emails).
- **Private Key**: Cryptographic secret (256-bit number) that authorizes cryptocurrency transactions. Equivalent to password + signature authority combined. Compromise = permanent fund loss (irreversible). Best practice: hardware wallet storage (never expose to internet-connected devices).
- **Hardware Wallet**: Physical device (Ledger, Trezor) that stores private keys offline and signs transactions in isolated environment, protecting against malware on host computers/phones. Gold standard for self-custody security (99%+ protection against remote attacks).
- **Blind Signing**: Approving blockchain transactions (especially complex smart contracts) without understanding the operations being authorized. Common cause of losses: users approve malicious token permissions, NFT transfers, DeFi contract exploits. Mitigation: transaction simulation, human-readable explanations.
- **Malware (Infostealer)**: Malicious software designed to extract sensitive data from infected devices. Crypto-specific variants target wallet interfaces, browser extensions, clipboard (replacing copied addresses), and keylogging seed phrase entry. Defense: hardware wallets, antivirus, OS updates, verified software downloads.
- **Address Poisoning**: Attack where scammer sends small-value transaction to victim from address visually similar to victim's legitimate address (same first/last characters), hoping victim copies wrong address from transaction history for future large transfer. Mitigation: verify full address character-by-character, use address book.
- **Smart Contract Permissions**: Blockchain-specific security concept where users grant dApps permission to transfer tokens on their behalf (ERC-20 "approve" function). Over-permissioning (unlimited approval) allows malicious contracts to drain wallets. Best practice: limited approvals, regular permission audits (revoke unused approvals).
- **Cold Storage**: Cryptocurrency stored offline (hardware wallet, paper wallet, air-gapped computer) with no internet connectivity, protecting against remote attacks. Contrast with hot wallets (online, convenient but higher risk). Institutional best practice: 90-95% cold storage, 5-10% hot for operations.
