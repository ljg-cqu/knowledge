## Glossary

- **Account abstraction (AA)**: Blockchain feature separating authentication from accounts, enabling smart contract wallets with programmable transaction logic
- **Bundler**: Infrastructure operator aggregating UserOperations into single on-chain transaction, earning fees for service
- **EIP-4337**: Ethereum Improvement Proposal defining account abstraction via UserOperation mempool without protocol changes, finalized March 2023
- **EIP-7702**: Proposed alternative AA approach allowing EOAs to set contract code temporarily, competing with EIP-4337
- **Entry point contract**: Smart contract processing UserOperations, validating signatures, executing transactions in EIP-4337 model
- **Externally Owned Account (EOA)**: Traditional blockchain account controlled by private key (e.g., MetaMask, hardware wallet)
- **Guardian system**: Social recovery mechanism where trusted contacts (guardians) can recover smart wallet access
- **Native account abstraction**: Built-in blockchain protocol AA support (zkSync, StarkNet) vs Ethereum's EIP-4337 meta-transaction approach
- **Paymaster**: Service sponsoring gas fees for users, enabling gasless transactions and alternative payment tokens
- **Session key**: Limited-permission temporary key for specific DApp interactions, reducing transaction approval friction
- **UserOperation**: Pseudo-transaction object in EIP-4337 model containing intent, signature, gas parameters, processed by bundlers
