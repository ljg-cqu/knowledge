## Glossary

- **Dependency tree**: Hierarchical structure of package dependencies, direct dependencies and transitive (indirect) dependencies creating chains 5-10 levels deep
- **Maintainer account compromise**: Attacker gains access to npm or GitHub credentials of package maintainer enabling malicious version publishing
- **npm (Node Package Manager)**: JavaScript package registry with 3M+ packages and 15B+ weekly downloads, dominant distribution for Web3 SDKs
- **Personal Access Token (PAT)**: GitHub authentication credential targeted by Shai-Hulud attack enabling repository access and package publishing
- **SBOM (Software Bill of Materials)**: Comprehensive inventory of software dependencies in CycloneDX or SPDX format enabling transparency
- **Shai-Hulud**: Self-replicating worm targeting npm ecosystem September-November 2025, named after sandworms in Dune, compromised 500+ packages
- **Supply chain attack**: Compromise of software distribution chain (package registries, build systems, dependencies) to inject malicious code affecting downstream users
- **Transitive dependency**: Indirect dependency through dependency chains (A depends on B, B depends on C, so A has transitive dependency on C)
- **Typosquatting**: Malicious packages using names similar to legitimate packages (ethersjs vs ethers.js) to trick developers into installing
- **Web3 provider**: JavaScript API for blockchain interaction (window.ethereum, ethers.providers) enabling wallet operations from web applications
- **Zero-day malicious code**: Previously unknown malicious code not yet in vulnerability databases, undetectable by signature-based tools
