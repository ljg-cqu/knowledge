## Glossary

- **Behavioral Biometrics**: AI tracking user habits (typing speed, mouse movements, usage patterns) to distinguish legitimate user from attacker even with correct credentials
- **Decision Tree**: AI model using rule-based thresholds (transaction >$50K = flag, new device + large withdrawal = freeze) for fraud detection
- **Enhanced Due Diligence (EDD)**: Regulatory process requiring additional KYC/AML verification for flagged accounts, 30-90 days to complete
- **Explainable AI (XAI)**: AI system providing human-readable justification for decisions ("transaction flagged due to deviation from baseline behavior")
- **False Negative**: Fraudulent transaction incorrectly classified as legitimate (missed fraud detection), regulatory penalties $10M-$100M
- **False Positive**: Legitimate transaction/user incorrectly flagged as suspicious, causes wrongful account freeze, 35-45% of frozen accounts industry-wide
- **Neural Network**: AI model using pattern recognition comparing user behavior vs population baseline for anomaly detection
- **Tiered Risk Scoring**: Fraud detection assigning low/medium/high risk levels with graduated responses (monitoring, withdrawal limits, immediate freeze)
- **True Positive**: Fraudulent transaction correctly detected and blocked (successful fraud prevention), >95% accuracy industry-wide
- **Whitelisting**: User pre-registration of trusted withdrawal addresses, large transactions to whitelisted addresses skip fraud detection
- **Zero-Sum Trade-Off**: Security vs UX dilemma where reducing false positives 20-30% increases fraud 50-100% costing $50-200M annually
