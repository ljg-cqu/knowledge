## Glossary

- **BIP-32**: Bitcoin Improvement Proposal 32 - Hierarchical Deterministic (HD) wallet standard enabling derivation of multiple child keys from single master seed; specific to secp256k1 curve (ECDSA), incompatible with ed25519 (EdDSA).
- **BIP-44**: Bitcoin Improvement Proposal 44 - Multi-coin HD wallet derivation standard using path m/44'/coin_type'/account'/change/address_index; coin_type registry assigns unique IDs per blockchain (e.g., 0 Bitcoin, 60 Ethereum).
- **Cross-chain**: Operations involving multiple blockchains; includes atomic swaps (exchange assets across chains), cross-chain messaging (IBC, XCMP), bridges (wrap/unwrap assets), unified custody.
- **EdDSA (Edwards-curve Digital Signature Algorithm)**: Signature scheme using twisted Edwards curves; ed25519 variant used by Solana, Cardano, Polkadot; incompatible with ECDSA secp256k1, requires separate MPC implementation.
- **EIP-2333**: Ethereum Improvement Proposal 2333 - HD wallet derivation for BLS12-381 signatures (Ethereum 2.0 validators); incompatible with BIP-32 (secp256k1) and SLIP-0010 (ed25519).
- **EVM-compatible**: Blockchains supporting Ethereum Virtual Machine (Polygon, Arbitrum, Optimism, BSC, Avalanche C-Chain, 40+ chains); use ECDSA secp256k1 signatures + Ethereum transaction formats, MPC key reuse possible.
- **HTLC (Hashed Timelock Contract)**: Cryptographic primitive enabling atomic swaps; uses hash preimage + timelock (e.g., Bitcoin OP_CHECKLOCKTIMEVERIFY, Ethereum smart contracts) to exchange assets across chains without trusted intermediary.
- **Multi-curve MPC**: Threshold signature protocol enabling single key generation ceremony → derive keys for multiple elliptic curves (secp256k1 ECDSA, ed25519 EdDSA, BLS12-381 BLS); active cryptographic research, production implementations limited.
- **RLP (Recursive Length Prefix)**: Ethereum transaction serialization format; encodes nonce, gasPrice, gasLimit, to, value, data, v/r/s into compact byte array for hashing and signing.
- **Schnorr Signature**: Signature scheme over secp256k1 (Bitcoin Taproot BIP-340), ed25519 (Polkadot SR25519 variant), or other curves; linearity enables signature aggregation, threshold adaptation simpler than ECDSA.
- **SLIP-0010**: Satoshi Labs Improvement Proposal 10 - Extension of BIP-32 HD wallet derivation to ed25519 curves; enables Solana/Cardano derivation from seed phrase but incompatible with BIP-32 secp256k1 derivation.
- **UTXO (Unspent Transaction Output)**: Bitcoin transaction model where outputs from previous transactions become inputs to new transactions; contrast with account model (Ethereum, Solana) using balances + nonces.
