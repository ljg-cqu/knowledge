## Problem Statement

Affects 500+ enterprises deploying MPC wallets, 100+ custody providers, $50K-$200K annual operational costs, 2-8 hour procedures per refresh cycle, potential service disruptions, human error risks from complex manual processes.
   
   - **Background and current situation**:

MPC key refresh innovation: Threshold cryptography enables "proactive security" where key shares can be refreshed without changing the underlying master key or blockchain address [Web Search: Stackup, 2025]. Address continuity: MPC key refresh maintains blockchain address (Ethereum 0x..., Bitcoin bc1...) because master key unchanged, only share distribution rotated; enables seamless security upgrades without user-facing changes [Web Search: Blockdaemon, 2024]. ## Glossary

- **Additive refresh**: Key refresh technique adding random values summing to zero across shares (share_i_new = share_i_old + random_i, Σrandom_i = 0); simpler than full DKG, provides proactive security without changing master key.
- **Atomic cutover**: Zero-downtime refresh technique where new key shares generated while old shares remain active, followed by instantaneous (<10s) switch to new shares minimizing service disruption.
- **Audit trail**: Compliance documentation recording every step of key refresh (timestamp, participants, cryptographic attestations, verification results, approvals) for 5-7 year regulatory retention.
- **Distributed Key Generation (DKG)**: Cryptographic protocol enabling multiple parties to jointly generate key shares without any party learning the complete key; requires 4-9 communication rounds, 5-30 minutes computation per round.
- **Feldman's Verifiable Secret Sharing (VSS)**: Cryptographic scheme enabling secret sharing with verification (participants can confirm shares are consistent without exposing secret); supports resharing (generating new shares for same master key).
- **Proactive security**: Security model where key shares are periodically refreshed to prevent gradual compromise; even if n-1 shares leaked over time, refreshed shares make historical leaks useless.
- **Split-phase protocol**: Zero-downtime refresh approach generating new key shares (phase 1) while old shares remain active for signing (phase 2), followed by atomic cutover (<10s) to new shares.
- **Static address**: Blockchain address (Ethereum 0x..., Bitcoin bc1...) remaining unchanged during key refresh; MPC refreshes shares without altering master key, maintaining address continuity for user deposits, smart contracts, audit trails.
- **Share destruction**: Security procedure securely wiping old key shares from all locations post-refresh (software cryptographic erasure, HSM key deletion, physical paper shredding, USB drive destruction) to prevent historical compromise.
- **Threshold signing**: Digital signature scheme requiring k-of-n parties cooperate to sign; prevents single point of failure, but coordination complexity multiplied during key refresh procedures.
- **Zero-knowledge proof**: Cryptographic proof enabling verification without revealing underlying secrets; used in key refresh to validate new shares are consistent with master key without exposing key or shares.
