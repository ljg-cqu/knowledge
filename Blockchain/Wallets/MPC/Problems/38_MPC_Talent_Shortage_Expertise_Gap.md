## Glossary

- **BitForge**: Critical vulnerability discovered in GG18/GG20 threshold ECDSA implementations (2023) affecting 15+ major MPC wallet providers, enabling key extraction attacks, demonstrating gap between protocol understanding and secure implementation.
- **CGGMP21**: UC Non-Interactive, Proactive, Threshold ECDSA protocol by Canetti, Gennaro, Goldfeder, Makriyannis, Peled (2021) with 4-round signing, state-of-the-art security; published 2021, production implementations appearing 2024-2025, demonstrating 3-4 year research-to-production lag.
- **FROST**: Flexible Round-Optimized Schnorr Threshold signatures; efficient threshold Schnorr protocol for non-ECDSA chains (Zcash, Taproot); published 2020, limited production use 2024-2025, demonstrating 4-5 year adoption lag.
- **GG18/GG20**: Threshold ECDSA protocols by Gennaro and Goldfeder (2018, 2020); widely deployed in first-generation MPC wallets but vulnerable to key extraction attacks (BitForge 2023), illustrating implementation security challenges.
- **MPC (Multi-Party Computation)**: Cryptographic technique enabling threshold signatures requiring k-of-n key shares without reconstructing complete private key; requires specialized expertise in threshold cryptography, zero-knowledge proofs, secure implementation practices.
- **Threshold cryptography**: Branch of cryptography enabling distributed key generation and signing across n parties with k-of-n threshold; foundational to MPC wallets but requiring graduate-level mathematical knowledge (number theory, algebraic structures, computational hardness).
- **Zero-knowledge proofs (ZKP)**: Cryptographic proofs enabling verification without revealing underlying data (zk-SNARKs, Bulletproofs, STARKs); increasingly required for privacy-preserving MPC architectures, representing frontier cryptographic research.
