## Glossary

- **Air-gapped system**: Computer or network physically isolated from unsecured networks (no internet, no wireless connectivity); used in highest-security environments to prevent remote attacks, complicating MPC's multi-party coordination.
- **Chain-of-custody**: Documentation tracking physical device movement through secure transfer process; required for USB devices crossing air-gap to prevent malware introduction or tampering.
- **Common Criteria (CC)**: International standard (ISO/IEC 15408) for computer security certification; EAL 4+ (Evaluation Assurance Level) required for defense/intelligence HSM deployments.
- **FIPS 140-3**: Federal Information Processing Standard defining cryptographic module security requirements; Level 3 (physical tamper-evidence) or Level 4 (tamper-active response) required for government/banking HSMs; certification requires 6-12 months testing.
- **HSM (Hardware Security Module)**: Tamper-proof physical device protecting cryptographic keys and performing signing operations; enterprise standard costing $50K-$300K per device, complicating MPC integration requiring share storage vs. complete keys.
- **Hybrid air-gapped architecture**: MPC configuration with 1 share in offline HSM (maximum security) and n-1 shares online (operational efficiency); balances security and performance, achieving <15 min signing vs. 30-90 min full air-gap.
- **Pre-signing**: MPC optimization generating signatures in advance (before transaction parameters finalized), reducing signing latency at transaction time from 2-15s to <100ms; limited applicability (requires predicting transaction details), useful for batch operations.
- **QR code protocol**: Air-gap crossing mechanism encoding MPC protocol messages as QR codes scanned by camera; limited to ~5KB per code, 4-round CGGMP20 requires ~20 scans (10 minutes at 30 sec per scan).
- **Remote attestation**: Cryptographic technique proving code integrity in secure enclave (Intel SGX, AMD SEV); enables verification that MPC share operations occur in trusted environment, but enclave vulnerabilities (Spectre, Meltdown) limit security guarantees.
- **Secure enclave**: Hardware-based isolated execution environment (Intel SGX, AMD SEV, ARM TrustZone) running code in encrypted memory inaccessible to OS; alternative to HSM for share protection, but faces ongoing vulnerabilities and 2-5× performance overhead.
- **Share-based architecture**: MPC key management model where no complete key exists; key mathematically split into shares, threshold coordination produces signatures without reconstructing key; conflicts with HSM design expecting complete keys.
- **Tamper-proof hardware**: Physical security measures in HSMs causing key destruction if breach detected (e.g., opening enclosure, voltage manipulation, temperature extremes); provides protection against physical attacks, a key advantage over software-based MPC shares.
- **Threshold signature**: Digital signature scheme requiring k-of-n parties to cooperate; prevents single point of failure, but coordination complexity increases dramatically in air-gapped environments (30-90 min signing vs. 2-15s online).
- **USB air-gap crossing**: Secure data transfer using physically verified USB devices; requires malware scanning, chain-of-custody documentation, device authentication; adds 2-5 minutes per crossing, more efficient than QR codes for larger data (>5KB).
