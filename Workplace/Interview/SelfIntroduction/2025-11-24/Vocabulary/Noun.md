## Blockchain Core Concepts

### **TVL (Total Value Locked)**
- **Meaning**: The total value of crypto assets deposited in a DeFi protocol
- **When to Use**: Discussing project scale, success metrics, or protocol security
- **Examples**:
  - "I led the uniBTC protocol development which achieved $4 million TVL"
  - "The DeFi protocols I managed had million-dollar TVL"

### **Restaking**
- **Meaning**: Using already-staked tokens to secure additional protocols and earn extra rewards
- **When to Use**: Explaining advanced staking mechanisms or protocol innovations
- **Examples**:
  - "I designed the uniBTC multi-chain BTC Restaking protocol"
  - "I integrated EigenLayer Restaking functionality for uniETH"

### **Liquid Staking**
- **Meaning**: A staking mechanism that issues tradeable derivative tokens representing staked assets
- **When to Use**: Discussing DeFi protocol design or capital efficiency improvements
- **Examples**:
  - "I developed uniIOTX as an IOTX Liquid Staking solution"
  - "I designed ETH Liquid Staking contracts integrated with EigenLayer"

### **Smart Contract**
- **Meaning**: Self-executing programs deployed on blockchain that automatically enforce agreements
- **When to Use**: Discussing development work, security concerns, or technical implementation
- **Examples**:
  - "I developed multiple production-grade Solidity smart contracts"
  - "I was responsible for smart contract auditing and security reviews"

### **Cross-chain**
- **Meaning**: Technology enabling communication and asset transfers between different blockchain networks
- **When to Use**: Explaining interoperability solutions or multi-chain deployments
- **Examples**:
  - "I designed the uniETH cross-chain transaction system"
  - "I solved core challenges in cross-chain communication and consistency"

### **Consensus Algorithm**
- **Meaning**: Mechanism by which blockchain nodes agree on the state of the distributed ledger
- **When to Use**: Discussing blockchain fundamentals or architectural decisions
- **Examples**:
  - "I have deep understanding of blockchain fundamentals and consensus algorithms"
  - "The consensus algorithm selection impacted our throughput requirements"

## Cryptography & Security

### **MPC (Multi-Party Computation)**
- **Meaning**: Cryptographic protocol allowing multiple parties to jointly compute a function while keeping inputs private
- **When to Use**: Discussing security architecture, key management, or asset protection
- **Examples**:
  - "I designed the MPC multi-party computation system for Lido-on-Avalanche"
  - "I implemented key splitting and multi-signature using MPC technology"

### **Multi-signature**
- **Meaning**: Security mechanism requiring multiple private keys to authorize a transaction
- **When to Use**: Explaining asset security measures or governance mechanisms
- **Examples**:
  - "I implemented key splitting and multi-signature securing substantial assets"
  - "The multi-signature system ensured decentralized key management"

### **Vulnerability**
- **Meaning**: Security weakness in code or system that could be exploited
- **When to Use**: Discussing code review, security audits, or quality improvements
- **Examples**:
  - "I audited and fixed critical security vulnerabilities in production contracts"
  - "I fixed 20+ security vulnerabilities and performance issues in Bedrock-DAO"

### **Audit**
- **Meaning**: Systematic examination of code or systems to identify bugs and security issues
- **When to Use**: Discussing quality assurance, security practices, or code review
- **Examples**:
  - "I was responsible for Bedrock-DAO contract auditing and testing"
  - "I conducted security audits for multiple DeFi protocols"

## Infrastructure & Architecture

### **Node**
- **Meaning**: A computer participating in a blockchain network, validating and relaying transactions
- **When to Use**: Discussing infrastructure, network operations, or system optimization
- **Examples**:
  - "I optimized large-scale node clusters for blockchain networks"
  - "I deployed validator node monitoring systems ensuring network security"

### **Validator**
- **Meaning**: A node that validates transactions and creates new blocks in proof-of-stake blockchains
- **When to Use**: Discussing staking protocols, network participation, or consensus mechanisms
- **Examples**:
  - "I deployed validator node monitoring system for the Polkadot parachain"
  - "The validator management system secured substantial staked assets"

### **Nonce**
- **Meaning**: A sequential number used to track transaction order from an Ethereum address
- **When to Use**: Discussing transaction management, concurrency issues, or system optimization
- **Examples**:
  - "I resolved nonce conflicts in multi-chain transaction systems"
  - "I developed a nonce tracker component solving transaction consistency issues"

### **Gas Fee**
- **Meaning**: Transaction cost paid to blockchain network for executing operations
- **When to Use**: Discussing cost optimization, user experience, or chain selection
- **Examples**:
  - "I designed a Substrate-based chain achieving zero Gas fee transactions"
  - "Gas fee optimization was critical for improving user adoption"

### **Microservice**
- **Meaning**: Architectural style structuring applications as collections of loosely coupled services
- **When to Use**: Discussing system architecture, scalability, or deployment strategy
- **Examples**:
  - "I built Docker-based microservice architecture for DeFi backends"
  - "The microservice design improved system maintainability and scalability"

## Specialized Technologies

### **EVM (Ethereum Virtual Machine)**
- **Meaning**: Runtime environment for executing smart contracts on Ethereum and compatible chains
- **When to Use**: Discussing smart contract deployment, chain compatibility, or ecosystem choice
- **Examples**:
  - "I have cross-ecosystem experience in major platforms including EVM"
  - "I designed an EVM-compatible chain based on Substrate"

### **Parachain**
- **Meaning**: Independent blockchain running parallel to Polkadot's relay chain
- **When to Use**: Discussing Polkadot ecosystem or multi-chain architecture
- **Examples**:
  - "I developed a Rococo testnet parachain for the LABS platform"
  - "The parachain integrated IPFS distributed storage"

### **Substrate**
- **Meaning**: Modular framework for building custom blockchains in the Polkadot ecosystem
- **When to Use**: Explaining blockchain development, Polkadot projects, or custom chain design
- **Examples**:
  - "I designed an EVM-compatible chain based on Substrate"
  - "My Substrate experience includes parachain development and deployment"

### **IPFS (InterPlanetary File System)**
- **Meaning**: Distributed file storage protocol for decentralized content addressing
- **When to Use**: Discussing storage solutions, decentralization, or data availability
- **Examples**:
  - "I integrated IPFS distributed storage in multiple projects"
  - "The NFT system integrated with IPFS for decentralized content storage"

### **Agent**
- **Meaning**: An autonomous AI system that can perceive, decide, and act to achieve goals
- **When to Use**: Discussing AI integration, automation, or intelligent systems
- **Examples**:
  - "I built distributed Agent systems using AWS Bedrock Claude"
  - "The AI Agent platform implemented automated trading strategy management"

## Additional Context

### **When Discussing Projects**
Use these nouns to demonstrate:
- **Technical depth**: MPC, consensus algorithm, nonce management
- **Scale**: TVL, node cluster, multi-chain deployment
- **Innovation**: Restaking, liquid staking, plugin architecture
- **Leadership**: Architecture, lifecycle, team management

### **Interview Tips**
1. **Quantify when possible**: "$4 million TVL", "12+ networks", "800-node cluster"
2. **Connect nouns to achievements**: "The MPC system secured substantial assets"
3. **Show problem-solving**: "I resolved nonce conflicts", "I fixed 20+ vulnerabilities"
4. **Demonstrate breadth**: Reference multiple tech stacks, frameworks, and ecosystems
