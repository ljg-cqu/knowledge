区块链安全密码开发工程师

职位详情

                            工作职责：
设计与实现基于 MPC 的钱包核心模块
包括密钥生成、密钥分片管理、签名协作协议、恢复流程等。
参与钱包签名协议的具体落地（如 GG18 / GG20 / FROST / Threshold ECDSA / Threshold EdDSA 等算法的工程实现和优化）。
负责交易签名环节的安全性、性能和稳定性优化，确保在移动端 / Web / 后端服务侧的可用性与低延迟表现。
与后端/安全团队一起构建风控、设备校验、多因子验证等安全策略，把“安全策略”产品化。
将底层密码学能力封装成 SDK / API，赋能内部产品和外部合作方集成。
参与钱包相关基础设施建设：例如账户抽象(AA)、Session Key、社交恢复、限额/审批流等体验层功能。
跟进主流公链生态（以太坊、EVM L2、BTC、Solana 等）的交易结构与签名标准，确保兼容性。

技术要求：
扎实的编程能力，熟练掌握至少一种系统级/密码学友好的语言：Rust / Go / C++ 。
有良好的代码抽象和工程化能力，能写可维护、可测试、可复用的模块。
密码学/安全相关背景，理解基本公钥密码学原理（ECDSA、Ed25519、椭圆曲线、哈希承诺、零知识证明的基本概念）。
对“多方计算 / 阈值签名 / 密钥分片 / 零单点托管风险”有实际理解，而不是只停留在概念层。
Web3 / 区块链经验
了解链上交易的生命周期：构造、签名、广播、上链、追踪。
熟悉至少一种主流链（Ethereum / BTC / Solana 任选其一）的交易结构和 RPC 交互。
安全意识
有过安全敏感系统的开发经验（钱包、风控、支付、资产托管、KMS、HSM 等），并理解攻击面与滥用场景。
