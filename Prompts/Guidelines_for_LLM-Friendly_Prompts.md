# Guidelines for LLM-Friendly Prompts

Optimize prompts for high-quality, precise outputs.

## Guidelines

### Foundation: Define the Task

1. **Context**: State scope, constraints, assumptions.
2. **Clarity**: Request clear, understandable explanations; avoid jargon without definition.
3. **Precision**: Use specific terms; avoid ambiguity; maintain consistent terminology.
4. **Relevance**: Stay on topic; exclude tangential information.

### Scope: What to Cover

5. **MECE**: Ensure complete, non-overlapping coverage.
6. **Sufficiency**: Ensure comprehensive coverage of all necessary aspects.
7. **Breadth**: Request multiple perspectives where appropriate.
8. **Depth**: Request thorough treatment with sufficient detail.

### Quality: Ensure Excellence

9. **Significance**: Prioritize important information; exclude trivial details.
10. **Concision**: Remove redundancy; include only essential information.
11. **Accuracy**: Verify factual correctness; cross-check information.
12. **Credibility & Reliability**: Prioritize authoritative, high-quality, tested sources; avoid low-quality or unproven content.
13. **Logic**: Demand coherent reasoning and sound structure.
14. **Risk/Value**: Assess risks, costs, and benefits; prioritize high-value, low-risk options; flag high-risk choices with mitigation strategies.
15. **Fairness**: Request balanced perspectives; acknowledge assumptions, limitations, biases, alternatives, counterarguments, trade-offs, and misconceptions.

### Format: How to Present

16. **Structure**: Use lists, tables, diagrams, formulas, code blocks.
17. **Output Format**: Request structured format with TOC linking to sections.

### Validation: Ensure Correctness

18. **Evidence**: Cite credible sources; flag uncertainty.
19. **Validation**: Request reasoning, self-review, error checks.
20. **Practicality**: Ensure actionable, implementable guidance.
21. **Success Criteria**: Define measurable, concrete outcomes.
