# Guidelines for LLM-Friendly Prompts

Optimize prompts for high-quality, precise outputs.

## Guidelines

### Foundation: Define the Task
1. **Context**: State scope, constraints, assumptions.
2. **Clarity**: Request clear, understandable explanations; avoid jargon without definition.
3. **Precision**: Use specific terms; avoid ambiguity; maintain consistent terminology.
4. **Relevance**: Stay on topic; exclude tangential information.

### Scope: What to Cover
5. **MECE**: Ensure complete, non-overlapping coverage.
6. **Sufficiency**: Ensure comprehensive coverage of all necessary aspects.
7. **Breadth**: Request multiple perspectives where appropriate.
8. **Depth**: Request thorough treatment with sufficient detail.

### Quality: Ensure Excellence
9. **Accuracy**: Verify factual correctness; cross-check information.
10. **Credibility & Reliability**: Prioritize authoritative, high-quality, tested sources; avoid low-quality or unproven content.
11. **Significance**: Prioritize important information; exclude trivial details.
12. **Logic**: Demand coherent reasoning and sound structure.
13. **Fairness**: Request balanced perspectives; acknowledge limitations, biases, and counterarguments.

### Format: How to Present
14. **Concision**: Remove redundancy; include only essential information.
15. **Structure**: Use lists, tables, diagrams, formulas, code blocks.
16. **Output Format**: Request structured format with TOC linking to sections.

### Validation: Ensure Correctness
17. **Evidence**: Cite credible sources; flag uncertainty.
18. **Validation**: Request reasoning, self-review, error checks.
19. **Practicality**: Ensure actionable, implementable guidance.
20. **Success Criteria**: Define measurable, concrete outcomes.
