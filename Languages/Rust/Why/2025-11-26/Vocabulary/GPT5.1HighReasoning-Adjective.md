# Adjective Vocabulary - GPT5.1HighReasoning

1. **Adjective**: pluggable (gradable, attributive)
   **Meaning**: 
   - Able to be inserted, attached, or integrated into a system; modular and interchangeable
   
   **Synonyms**: modular, interchangeable, swappable, flexible, extensible
   
   **Antonyms**: fixed, hardcoded, monolithic, inflexible
   
   **When to Use**: Technical contexts describing modular software architecture
   
   **When NOT to Use**: Non-technical contexts where "flexible" would be clearer
   
   **Common Phrases**: 
   - pluggable abstractions
   - pluggable architecture
   - pluggable components
   
   **Root Analysis**: plug + -able (capable of)
   
   **Etymology**: Modern formation from "plug" + "-able" suffix
   
   **Examples [Casual]**: 
   - The system is pluggable - you can swap out components easily.
   - I like that it's pluggable so I can use different backends.
   
   **Examples [Formal]**: 
   - Rust wants async abstractions to remain lightweight and pluggable.
   - The pluggable design allows for domain-specific optimizations.

1. **Adjective**: lightweight (comp: lighter-weight, gradable, attributive)
   **Meaning**: 
   - *(Technical sense)* Using minimal system resources; having low computational overhead
   - Not heavy or cumbersome in design
   
   **Synonyms**: minimal, efficient, lean, compact, streamlined
   
   **Antonyms**: heavyweight, bloated, resource-intensive, cumbersome
   
   **When to Use**: Technical contexts describing efficient software or systems
   
   **When NOT to Use**: When referring to actual physical weight
   
   **Common Phrases**: 
   - lightweight abstractions
   - lightweight runtime
   - lightweight process
   
   **Root Analysis**: light + weight (compound)
   
   **Etymology**: Modern English compound formation
   
   **Examples [Casual]**: 
   - This editor is super lightweight - starts up instantly.
   - We need a lightweight solution that won't slow things down.
   
   **Examples [Formal]**: 
   - Rust's async model uses lightweight state machines without runtime overhead.
   - Lightweight executors provide flexibility for embedded systems.

1. **Adjective**: explicit (gradable, attributive/predicative)
   **Meaning**: 
   - Stated clearly and in detail; leaving no room for confusion
   
   **Synonyms**: clear, direct, unambiguous, specific, overt
   
   **Antonyms**: implicit, vague, ambiguous, unclear
   
   **When to Use**: Technical and formal contexts requiring precision
   
   **When NOT to Use**: When subtlety or implication is preferred
   
   **Common Phrases**: 
   - explicit constraints
   - explicit type signatures
   - make explicit
   
   **Root Analysis**: ex (out) + plicare (to fold)
   
   **Etymology**: Latin → explicitus (unfolded, clearly stated)
   
   **Examples [Casual]**: 
   - Be explicit about what you need - don't make me guess.
   - The error message was explicit about what went wrong.
   
   **Examples [Formal]**: 
   - Combining ownership with executors forces explicit constraints in type signatures.
   - Explicit lifetime annotations clarify reference validity.

1. **Adjective**: opaque (gradable, predicative)
   **Meaning**: 
   - *(Figurative)* Not able to be understood; unclear or difficult to comprehend
   - *(Literal)* Not transparent or translucent
   
   **Synonyms**: unclear, obscure, cryptic, impenetrable, unintelligible
   
   **Antonyms**: transparent, clear, obvious, understandable
   
   **When to Use**: Technical contexts describing unclear messages or difficult-to-understand systems
   
   **When NOT to Use**: When describing physical transparency
   
   **Common Phrases**: 
   - opaque error messages
   - opaque implementation
   - opaque abstractions
   
   **Root Analysis**: from Latin opacus
   
   **Etymology**: Latin → opacus (shaded, dark)
   
   **Examples [Casual]**: 
   - The error messages are pretty opaque - I can't figure out what's wrong.
   - His explanation was opaque, full of jargon.
   
   **Examples [Formal]**: 
   - Opaque error messages can hinder developer productivity during debugging.
   - The abstraction remains intentionally opaque to hide implementation details.

1. **Adjective**: idiomatic (gradable, attributive)
   **Meaning**: 
   - *(Programming)* Following the natural or conventional patterns of a language
   - Using expressions natural to a native speaker or established practice
   
   **Synonyms**: natural, conventional, typical, standard, canonical
   
   **Antonyms**: unidiomatic, awkward, foreign, unconventional
   
   **When to Use**: Programming and language contexts describing natural or proper usage
   
   **When NOT to Use**: Casual contexts where "typical" or "standard" is clearer
   
   **Common Phrases**: 
   - idiomatic Rust
   - idiomatic code
   - idiomatic patterns
   
   **Root Analysis**: idiom + -atic (relating to)
   
   **Etymology**: Greek → idioma (peculiarity, property)
   
   **Examples [Casual]**: 
   - That's not very idiomatic - experienced developers would do it differently.
   - Try to write idiomatic code that follows community standards.
   
   **Examples [Formal]**: 
   - Insufficient developer education results in non-idiomatic Rust patterns.
   - Idiomatic error handling uses the Result type with the ? operator.

1. **Adjective**: baked-in (non-gradable, attributive)
   **Meaning**: 
   - Built into the fundamental structure; integral and unchangeable
   - Included as a permanent feature
   
   **Synonyms**: built-in, integrated, inherent, embedded, hardcoded
   
   **Antonyms**: external, separate, modular, optional
   
   **When to Use**: Technical contexts describing permanently integrated features
   
   **When NOT to Use**: Formal writing (it's somewhat colloquial)
   
   **Common Phrases**: 
   - baked-in runtime
   - baked-in assumptions
   - baked-in features
   
   **Root Analysis**: bake + in (phrasal verb converted to adjective)
   
   **Etymology**: Modern metaphorical extension from baking (permanently setting)
   
   **Examples [Casual]**: 
   - The tracking is baked-in - you can't turn it off.
   - Those assumptions are baked-in to the architecture.
   
   **Examples [Formal]**: 
   - Ergonomic async typically requires runtime support baked-in to the language.
   - The language cannot assume baked-in runtimes for embedded systems.

1. **Adjective**: agnostic (non-gradable, postpositive with complement)
   **Meaning**: 
   - *(Technical)* Independent of or neutral about a particular platform, system, or technology
   - Not committed to a particular viewpoint
   
   **Synonyms**: neutral, independent, platform-independent, flexible
   
   **Antonyms**: platform-specific, dependent, tied-to, coupled
   
   **When to Use**: Technical contexts describing technology independence
   
   **When NOT to Use**: Religious contexts (where it has theological meaning)
   
   **Common Phrases**: 
   - runtime-agnostic
   - platform-agnostic
   - database-agnostic
   
   **Root Analysis**: a (without) + gnostic (knowing)
   
   **Etymology**: Greek → agnostos (unknown, unknowable)
   
   **Examples [Casual]**: 
   - The tool is platform-agnostic - works on Windows, Mac, and Linux.
   - We want a database-agnostic approach so we can switch later.
   
   **Examples [Formal]**: 
   - Rust's runtime-agnostic async design exposes complexity explicitly in types.
   - A runtime-agnostic approach ensures portability across diverse execution environments.

1. **Adjective**: native (non-gradable, attributive)
   **Meaning**: 
   - *(Computing)* Designed for a specific platform or system architecture
   - Belonging to a person or thing by nature; innate
   
   **Synonyms**: indigenous, built-in, inherent, original, natural
   
   **Antonyms**: foreign, external, imported, added
   
   **When to Use**: Technical contexts describing platform-specific implementations
   
   **When NOT to Use**: Sensitive contexts involving people or cultures
   
   **Common Phrases**: 
   - native code
   - native compilation
   - native performance
   
   **Root Analysis**: from Latin nativus
   
   **Etymology**: Latin → nativus (innate, natural)
   
   **Examples [Casual]**: 
   - The app runs native on Mac - not a web wrapper.
   - Native apps are faster than cross-platform ones.
   
   **Examples [Formal]**: 
   - Rust's primary compilation target is safe, optimized native code.
   - Native compilation eliminates interpretation overhead at runtime.

1. **Adjective**: centric (non-gradable, used in compounds)
   **Meaning**: 
   - Having a specified thing as the most important element
   - Centered on or focused around something
   
   **Synonyms**: focused, centered, oriented, based
   
   **Antonyms**: peripheral, secondary, tangential
   
   **When to Use**: Technical and strategic contexts describing focus areas
   
   **When NOT to Use**: As standalone word (typically used in compounds)
   
   **Common Phrases**: 
   - ownership-centric
   - user-centric
   - data-centric
   
   **Root Analysis**: center + -ic (relating to)
   
   **Etymology**: Latin → centrum (center) + -ic
   
   **Examples [Casual]**: 
   - They're really customer-centric - always asking for feedback.
   - The design is mobile-centric, desktop is an afterthought.
   
   **Examples [Formal]**: 
   - Impedance mismatch between Rust's ownership-centric design and existing runtimes.
   - An ownership-centric approach requires explicit resource management.

1. **Adjective**: mainstream (non-gradable, attributive)
   **Meaning**: 
   - Belonging to or characteristic of the dominant trend in opinion, style, or activity
   - Widely accepted or practiced
   
   **Synonyms**: conventional, established, standard, dominant, prevalent
   
   **Antonyms**: alternative, niche, fringe, unconventional
   
   **When to Use**: Market and cultural contexts describing widespread adoption
   
   **When NOT to Use**: When describing cutting-edge or innovative practices
   
   **Common Phrases**: 
   - mainstream languages
   - mainstream adoption
   - mainstream developers
   
   **Root Analysis**: main + stream (compound)
   
   **Etymology**: Modern English compound (originally from "main stream" of thought)
   
   **Examples [Casual]**: 
   - Python is pretty mainstream now - everyone knows it.
   - That's not mainstream tech - most companies don't use it.
   
   **Examples [Formal]**: 
   - Rust compile times feel slower than mainstream languages like Go.
   - No mainstream language had successfully unified safety and control until Rust.
