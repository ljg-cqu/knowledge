Core Concepts: The Pillars of Rust

Ownership: The Foundation of Memory Safety

Rust's ownership system is a novel approach to memory management that ensures memory safety without the need for a garbage collector. This system is enforced by the compiler at compile time, preventing common memory-related bugs before code execution.

The Three Rules of Ownership

1
Single Owner: Each value in Rust has a single, unique owner responsible for managing its memory
2
Automatic Drop: When the owner goes out of scope, the value is automatically dropped and memory is freed
3
Ownership Transfer: Ownership can be transferred through "moving," preventing multiple owners

String Type Example

let s1 = String::from("hello");
let s2 = s1;  // Ownership moves from s1 to s2

// println!("{}", s1); // Error: s1 is no longer valid
println!("{}", s2); // Valid: s2 now owns the String

This demonstrates how ownership transfer prevents double-free errors while avoiding unnecessary data copying. 16

Borrowing: Accessing Data Without Ownership

Borrowing allows temporary access to data without taking ownership, enabling safe and efficient code through references.

Immutable References (&T)

let s = String::from("hello");
let len = calculate_length(&s);

fn calculate_length(s: &String) -> usize {
    s.len()
}

Read-only access that preserves the original owner

Mutable References (&mut T)

let mut s = String::from("hello");
change(&mut s);

fn change(s: &mut String) {
    s.push_str(", world");
}

Allows modification while maintaining ownership rules

The Borrowing Rule

You can have either one mutable reference or any number of immutable references, but not both simultaneously.

This rule prevents data races at compile time, making concurrent programming safer. 17

Lifetimes: Ensuring Reference Validity

Lifetimes are a compile-time mechanism that tracks the scope for which references are valid, preventing dangling references.

Lifetime Annotations in Functions

fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}

The 'a lifetime annotation ensures the returned reference is valid as long as both input references are valid.

Lifetime Elision

Rust can infer lifetimes in many common situations, making code more ergonomic without sacrificing safety. 

fn first_word(s: &str) -> &str {
    // Lifetime elision rules apply
}

'static Lifetime

The 'static lifetime represents the entire program duration, typically used for string literals. 

let s: &'static str = "Hello, world!";
// Stored in program binary, always valid
