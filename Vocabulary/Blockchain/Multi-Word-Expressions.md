# Blockchain Vocabulary: Multi-Word Expressions

1. **Expression**: smart contract (type: collocation)
   **Meaning**: 
   - Self-executing code deployed on a blockchain that automatically enforces agreement terms when conditions are met
   - *Fixedness*: strong (fixed partnership—not "intelligent contract" or "clever contract")
   
   **Synonyms**: autonomous program, self-executing agreement, blockchain code, programmable contract, on-chain logic
   
   **Antonyms**: traditional contract, paper agreement, manual agreement, off-chain contract
   
   **When to Use**: When discussing programmable blockchain functionality, DeFi protocols, automated execution, or decentralized applications. Core blockchain concept.
   
   **When NOT to Use**: Don't use for simple transactions without logic. Avoid when referring to legal contracts (smart contracts aren't legally binding by default). Not appropriate for off-chain code.
   
   **Common Variations**: 
   - smart contract code
   - deploy a smart contract
   - smart contract platform
   - smart contract vulnerability
   
   **Why This Partnership**: 
   - *For collocations*: "Smart" conveys automation and self-execution without human intervention; "contract" indicates agreement enforcement. Coined by Nick Szabo (1994) to describe programmable transaction protocols.
   
   **Root Analysis**: "smart" (automated/intelligent) + "contract" (agreement)
   
   **Examples [Casual]**: 
   - Smart contracts are like vending machines—put money in, automatically get something out.
   - You can create a smart contract that pays out rewards automatically.
   
   **Examples [Formal]**: 
   - Smart contracts enable trustless execution of complex multi-party agreements without intermediaries.
   - Ethereum smart contracts are Turing-complete programs executing deterministically across network nodes.

1. **Expression**: proof of work (type: fixed phrase)
   **Meaning**: 
   - A consensus mechanism requiring computational effort to validate transactions and create blocks, used by Bitcoin
   - *Fixedness*: strong (standardized technical term—not "evidence of work" or "work proof")
   
   **Synonyms**: PoW, computational consensus, mining-based consensus, hash-based consensus
   
   **Antonyms**: proof of stake, proof of authority, delegated proof of stake, PoS
   
   **When to Use**: When discussing Bitcoin's consensus mechanism, mining, or energy-intensive blockchain validation. Fundamental blockchain concept.
   
   **When NOT to Use**: Don't use for non-mining consensus mechanisms like proof of stake. Avoid confusing with "proof of work" in non-blockchain contexts.
   
   **Common Variations**: 
   - proof-of-work mining
   - PoW consensus
   - proof of work algorithm
   - proof of work blockchain
   
   **Why This Partnership**: 
   - *For collocations*: Technical term describing cryptographic puzzles that "prove" computational "work" was performed. Introduced in cryptocurrency context by Bitcoin (2008).
   
   **Root Analysis**: proof (evidence) + of (possession/characteristic) + work (computational effort)
   
   **Examples [Casual]**: 
   - Bitcoin uses proof of work, which is why miners need all that expensive hardware.
   - Proof of work is secure but uses a ton of electricity.
   
   **Examples [Formal]**: 
   - Proof of work achieves Sybil resistance by requiring costly computational expenditure for block production.
   - The proof of work difficulty adjustment algorithm maintains consistent block times despite hashrate fluctuations.

1. **Expression**: private key (type: collocation)
   **Meaning**: 
   - A secret cryptographic value that proves ownership and enables signing transactions to spend cryptocurrency
   - *Fixedness*: strong (standardized cryptographic term—not "secret key" or "personal key" in this context)
   
   **Synonyms**: secret key, signing key, wallet key, cryptographic secret
   
   **Antonyms**: public key, public address, shared key
   
   **When to Use**: When discussing cryptocurrency security, wallet control, transaction signing, or key management. Critical security concept.
   
   **When NOT to Use**: Don't confuse with passwords (private keys are cryptographic, not chosen phrases). Avoid exposing private keys—they should remain secret.
   
   **Common Variations**: 
   - private key security
   - lose your private key
   - private key management
   - private/public key pair
   
   **Why This Partnership**: 
   - *For collocations*: "Private" indicates secrecy and non-disclosure; "key" refers to cryptographic value unlocking access. Standard public-key cryptography terminology (1970s+).
   
   **Root Analysis**: private (secret/personal) + key (unlocking mechanism)
   
   **Examples [Casual]**: 
   - If you lose your private key, you lose access to your crypto forever—there's no recovery.
   - Never share your private key with anyone—it's like giving away your bank account.
   
   **Examples [Formal]**: 
   - Private keys generate digital signatures proving transaction authorization without revealing the key itself.
   - Hierarchical deterministic wallets derive multiple private keys from a single seed phrase following BIP-32 standards.

1. **Expression**: gas fee (type: collocation)
   **Meaning**: 
   - The transaction cost paid on Ethereum and similar blockchains for computational resources and network usage
   - *Fixedness*: strong (Ethereum-specific term—not "gas cost" typically, though sometimes used)
   
   **Synonyms**: transaction fee, network fee, execution cost, gas cost, processing fee
   
   **Antonyms**: free transaction, zero-cost execution
   
   **When to Use**: When discussing Ethereum transaction costs, network congestion, or smart contract execution expenses. Ethereum-specific terminology.
   
   **When NOT to Use**: Don't use for Bitcoin transaction fees (measured differently). Avoid when discussing blockchains without gas models.
   
   **Common Variations**: 
   - high gas fees
   - gas fee optimization
   - gas fee spike
   - pay gas fees
   
   **Why This Partnership**: 
   - *For collocations*: "Gas" metaphorically represents computational fuel (Ethereum, 2015); "fee" indicates cost paid. Natural pairing following the gas metaphor.
   
   **Root Analysis**: gas (computational unit) + fee (payment/cost)
   
   **Examples [Casual]**: 
   - Gas fees are insane right now—wait until late at night when it's cheaper.
   - This transaction failed because I didn't set the gas fee high enough.
   
   **Examples [Formal]**: 
   - Gas fees fluctuate based on network demand, creating market-driven resource allocation for block space.
   - EIP-1559 reformed gas fee mechanisms through base fees and priority tips to improve predictability.

1. **Expression**: double spend (type: fixed phrase/verb)
   **Meaning**: 
   - The act of spending the same cryptocurrency unit more than once, which blockchain consensus prevents
   - *Fixedness*: strong (standard blockchain security term—not "duplicate spending")
   
   **Synonyms**: double spending, spending twice, duplicate transaction, replay attack (related)
   
   **Antonyms**: single spend, unique transaction, non-replayable transaction
   
   **When to Use**: When discussing blockchain security, consensus mechanisms, or transaction finality. Core problem blockchain solves.
   
   **When NOT to Use**: Don't use for legitimate multiple transactions. Avoid confusing with accidental duplicate submissions (different concept).
   
   **Common Variations**: 
   - double-spend attack
   - prevent double spending
   - double spend problem
   - protection against double spends
   
   **Why This Partnership**: 
   - *For collocations*: "Double" indicates duplication/repetition; "spend" refers to transaction execution. Fixed term from digital currency literature (1990s+).
   
   **Root Analysis**: double (twice/duplicate) + spend (use currency)
   
   **Examples [Casual]**: 
   - Blockchain prevents double spending—you can't send the same Bitcoin to two different people.
   - Without consensus, someone could double spend their coins and cheat the system.
   
   **Examples [Formal]**: 
   - Byzantine fault-tolerant consensus mechanisms prevent double-spend attacks by requiring supermajority agreement on transaction ordering.
   - The double-spend problem historically prevented digital currency viability before blockchain consensus solutions.

1. **Expression**: mining pool (type: collocation)
   **Meaning**: 
   - A group of miners who combine computational resources to increase probability of finding blocks and share rewards
   - *Fixedness*: strong (standard mining term—not "miner group" or "hash pool")
   
   **Synonyms**: mining collective, hash pool, miner consortium, collaborative mining, pool mining
   
   **Antonyms**: solo mining, individual mining, independent mining
   
   **When to Use**: When discussing proof-of-work mining strategies, reward distribution, or mining centralization. Common in Bitcoin and PoW contexts.
   
   **When NOT to Use**: Don't use for proof-of-stake validators (different mechanism). Avoid confusing with liquidity pools (DeFi concept).
   
   **Common Variations**: 
   - join a mining pool
   - mining pool operator
   - largest mining pools
   - pool mining rewards
   
   **Why This Partnership**: 
   - *For collocations*: "Mining" indicates proof-of-work activity; "pool" conveys resource aggregation and shared rewards. Natural collocation from mining metaphor.
   
   **Root Analysis**: mining (PoW validation) + pool (collective resource)
   
   **Examples [Casual]**: 
   - Most miners join mining pools because solo mining rarely finds blocks anymore.
   - Mining pools split the rewards based on how much hash power you contribute.
   
   **Examples [Formal]**: 
   - Mining pool concentration raises centralization concerns, as few pools control majority network hashrate.
   - Pool mining reduces variance in reward distribution, providing more consistent income for participants.

1. **Expression**: decentralized finance (type: collocation/compound)
   **Meaning**: 
   - Financial services built on blockchain networks without traditional intermediaries, using smart contracts
   - *Fixedness*: strong (established industry term—abbreviated as DeFi)
   
   **Synonyms**: DeFi, open finance, permissionless finance, blockchain finance, crypto finance
   
   **Antonyms**: traditional finance, TradFi, centralized finance, CeFi, conventional banking
   
   **When to Use**: When discussing lending, trading, or financial services on blockchain platforms. Major blockchain use case category.
   
   **When NOT to Use**: Don't use for centralized crypto services (use "CeFi"). Avoid when discussing traditional fintech that doesn't use blockchain.
   
   **Common Variations**: 
   - DeFi protocol
   - DeFi platform
   - decentralized finance ecosystem
   - DeFi lending
   
   **Why This Partnership**: 
   - *For collocations*: "Decentralized" indicates blockchain-based architecture without intermediaries; "finance" specifies the application domain. Emerged ~2018-2019 as category term.
   
   **Root Analysis**: decentralized (distributed) + finance (financial services)
   
   **Examples [Casual]**: 
   - DeFi lets you lend, borrow, and trade crypto without banks or brokers.
   - Decentralized finance apps give you way higher interest rates than traditional savings accounts.
   
   **Examples [Formal]**: 
   - Decentralized finance protocols utilize smart contracts to provide lending, trading, and derivative services without custodial intermediaries.
   - DeFi's composability enables protocol integration, creating novel financial primitives impossible in traditional systems.

1. **Expression**: blockchain trilemma (type: fixed phrase)
   **Meaning**: 
   - The challenge of simultaneously achieving decentralization, security, and scalability—typically only two are fully achievable
   - *Fixedness*: strong (established theoretical concept—coined by Vitalik Buterin)
   
   **Synonyms**: scalability trilemma, three-way trade-off, DCS trilemma
   
   **Antonyms**: (no direct antonym—concept describes inherent trade-offs)
   
   **When to Use**: When discussing blockchain design trade-offs, limitations, or why different chains make different choices. Important theoretical framework.
   
   **When NOT to Use**: Don't use when only discussing two properties. Avoid claiming any solution has "solved" the trilemma (controversial). Not applicable to non-blockchain systems.
   
   **Common Variations**: 
   - solve the blockchain trilemma
   - trilemma trade-offs
   - addressing the trilemma
   - trilemma constraints
   
   **Why This Partnership**: 
   - *For idioms*: "Blockchain" specifies domain; "trilemma" indicates three-way impossible choice. Coined by Vitalik Buterin (2017) describing fundamental blockchain constraints.
   
   **Root Analysis**: blockchain (technology) + trilemma (three-way problem)
   
   **Examples [Casual]**: 
   - The blockchain trilemma means you can't have everything—fast, cheap, and decentralized at once.
   - Bitcoin chose security and decentralization but sacrificed scalability.
   
   **Examples [Formal]**: 
   - The blockchain trilemma posits fundamental trade-offs between decentralization, security, and throughput scalability.
   - Layer 2 solutions attempt to address the trilemma by moving computation off-chain while leveraging Layer 1 security.

1. **Expression**: cold storage (type: collocation)
   **Meaning**: 
   - Cryptocurrency storage method where private keys are kept completely offline, away from internet connectivity
   - *Fixedness*: strong (established security term—contrasts with "hot wallet")
   
   **Synonyms**: offline storage, cold wallet, hardware wallet (specific type), air-gapped storage
   
   **Antonyms**: hot wallet, hot storage, online wallet, connected wallet
   
   **When to Use**: When discussing cryptocurrency security, long-term holding, or protection against hacking. Best practice for significant holdings.
   
   **When NOT to Use**: Don't use for wallets that occasionally connect to internet. Avoid when discussing actively traded funds (use "hot wallet").
   
   **Common Variations**: 
   - keep in cold storage
   - cold storage wallet
   - move to cold storage
   - cold storage security
   
   **Why This Partnership**: 
   - *For collocations*: "Cold" metaphorically indicates dormant/inactive state (like refrigeration); "storage" indicates safekeeping. Contrasts with "hot" (active/online).
   
   **Root Analysis**: cold (inactive/offline) + storage (safekeeping)
   
   **Examples [Casual]**: 
   - For serious money, use cold storage—never leave large amounts on exchanges.
   - I keep 90% of my crypto in cold storage and only a little in hot wallets for trading.
   
   **Examples [Formal]**: 
   - Cold storage eliminates remote attack vectors by maintaining private keys on air-gapped devices.
   - Institutional custody solutions employ multi-signature cold storage with geographically distributed key shards.

1. **Expression**: hash rate (type: collocation)
   **Meaning**: 
   - The computational power used in mining, measured in hashes computed per second
   - *Fixedness*: strong (standard technical metric—not "hashing speed" typically)
   
   **Synonyms**: hashing power, mining power, computational power, network hashrate, hash power
   
   **Antonyms**: low hashrate, reduced mining power, computational weakness
   
   **When to Use**: When discussing mining difficulty, network security, or proof-of-work blockchain strength. Key security and performance metric.
   
   **When NOT to Use**: Don't use for proof-of-stake networks (different metrics). Avoid confusing with transaction throughput (different measurement).
   
   **Common Variations**: 
   - network hash rate
   - total hashrate
   - mining hashrate
   - hashrate distribution
   
   **Why This Partnership**: 
   - *For collocations*: "Hash" refers to cryptographic function computations; "rate" indicates speed/frequency. Natural technical measurement pairing.
   
   **Root Analysis**: hash (cryptographic output) + rate (frequency/speed)
   
   **Examples [Casual]**: 
   - Bitcoin's hash rate is at all-time highs, making it more secure than ever.
   - When hash rate drops, mining difficulty adjusts down to keep block times consistent.
   
   **Examples [Formal]**: 
   - Network hash rate serves as a proxy for proof-of-work security, as attacks require majority computational power.
   - Hash rate fluctuations correlate with mining profitability, electricity costs, and hardware availability.
