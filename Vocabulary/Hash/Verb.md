1. **Verb**: hash
   **Meaning**: 
   - (Transitive) To process data through a hash function to produce a fixed-size output value
   
   **Synonyms**: digest, compute hash, generate hash, checksum
   
   **Antonyms**: decrypt, reverse, unhash (not standard, but conceptually opposite)
   
   **When to Use**: Technical discussions about applying hash functions, password storage, data integrity verification
   
   **When NOT to Use**: When referring to encryption (which is reversible) or other non-hash transformations
   
   **Common Phrases**: 
   - hash the password
   - hash the data
   - hash the file
   - hash and salt
   - hash before storing
   
   **Root Analysis**: hash (chop, mix) - from culinary term
   
   **Etymology**: French hacher (to chop) → hash (1650s culinary) → hash (1970s computing verb)
   
   **Examples [Casual]**: 
   - Make sure to hash all passwords before saving them to the database.
   - Just hash the file and compare the values.
   
   **Examples [Formal]**: 
   - The system hashes sensitive credentials using SHA-256 prior to persistent storage.
   - Authentication protocols hash user inputs to prevent plaintext transmission.

1. **Verb**: compute

   - (Transitive) To calculate or determine using mathematical or algorithmic processes
   
   **Synonyms**: calculate, determine, process, generate, derive
   
   **Antonyms**: guess, estimate (non-precise methods)
   
   **When to Use**: Technical contexts involving calculations, algorithms, hash generation, mathematical operations
   
   **When NOT to Use**: Simple or non-technical contexts where "calculate" or "figure out" would be clearer
   
   **Common Phrases**: 
   - compute the hash
   - compute the digest
   - compute the checksum
   - compute efficiently
   - compute the value
   
   **Root Analysis**: com (together) + putare (to reckon, calculate)
   
   **Etymology**: Latin computare (to calculate) → compute (17th century)
   
   **Examples [Casual]**: 
   - The system computes the hash value in milliseconds.
   - Let me compute the checksum to verify the download.
   
   **Examples [Formal]**: 
   - The algorithm computes cryptographic digests with O(n) time complexity.
   - Consensus nodes compute hash-based proofs to validate block integrity.

1. **Verb**: encrypt
   **Meaning**: 
   - (Transitive) To convert data into coded form using an algorithm, making it unreadable without a decryption key (Note: different from hashing)
   
   **Synonyms**: encode, cipher, scramble, encipher
   
   **Antonyms**: decrypt, decipher, decode, unencrypt
   
   **When to Use**: Discussing reversible data protection, confidentiality, secure communication (distinct from hashing)
   
   **When NOT to Use**: When discussing one-way hash functions (use "hash" instead)
   
   **Common Phrases**: 
   - encrypt the data
   - encrypt with a key
   - encrypt and decrypt
   - encrypt for security
   - encrypt before transmission
   
   **Root Analysis**: en (in, into) + crypt (hidden)
   
   **Etymology**: Greek kryptos (hidden) → encrypt (20th century)
   
   **Examples [Casual]**: 
   - Encrypt your files before uploading them to the cloud.
   - Don't confuse encryption with hashing—encryption is reversible.
   
   **Examples [Formal]**: 
   - The protocol encrypts sensitive data using AES-256 prior to network transmission.
   - Unlike hash functions, encryption algorithms permit data recovery through decryption keys.

1. **Verb**: protect
   **Meaning**: 
   - (Transitive) To keep data safe from unauthorized access, tampering, or compromise using hashing
   
   **Synonyms**: secure, safeguard, defend, shield, guard
   
   **Antonyms**: expose, endanger, compromise, reveal
   
   **When to Use**: Security contexts, discussing protective measures, password security, data protection
   
   **When NOT to Use**: When discussing confidentiality (use "encrypt" instead, as hashing doesn't hide data)
   
   **Common Phrases**: 
   - protect passwords
   - protect with hashing
   - protect integrity
   - protect against attacks
   - protect sensitive data
   
   **Root Analysis**: pro (before, in front) + tect (cover)
   
   **Etymology**: Latin protegere (to cover in front) → protect (16th century)
   
   **Examples [Casual]**: 
   - Hash functions protect passwords by making them impossible to reverse.
   - We protect our data integrity using cryptographic hashes.
   
   **Examples [Formal]**: 
   - Cryptographic hash functions protect authentication systems against credential theft.
   - The protocol protects transaction integrity through hash-based immutability mechanisms.
